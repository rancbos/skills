# Step 0：数据准备与校验（强制前置）

> **核心原则**：**一切分析必须建立在可信数据之上。单一信源 = 错误风险。**
> **输入**：股票代码 + 市场（A/HK/US）→ **输出**：step0_data.json（供 Step 1-6 使用）

**💾 检查点状态检查**（执行前必做，强制）：
1. 执行以下命令检查检查点：
```bash
CHECKPOINT_DIR="/root/data/.checkpoints/{股票代码}_{YYYYMMDD}"
cat "$CHECKPOINT_DIR/checkpoint.json" 2>/dev/null || echo '{"status":"no_checkpoint"}'
```
2. 如果存在 checkpoint.json 且 `status=completed` → 读取报告路径，直接输出已有报告
3. 如果存在 checkpoint.json 且 `status=in_progress` → 读取 `last_step`，从 `last_step+1` 继续
   - **必须读取并复用**：step0_data.json（股价/财务/行业数据）、step1_quality.json（量化评分基线）
   - **必须补充**：web search 获取最新定性数据（管理层/事件/竞争对手）
   - **校验差异**：检查点数据 vs 当前数据，若时间差>7天需更新股价和最新季报
4. 如果 checkpoint.json 不存在但目录存在 → 从 Step 0 开始，忽略残留文件
5. 如果目录不存在 → 从 Step 0 开始
6. 如果任何 JSON 文件格式错误 → 忽略该检查点，从对应步骤重新执行

**💾 检查点写入模板**（每步结束时必须执行）：
```bash
# 写入当前步骤检查点
CHECKPOINT_DIR="/root/data/.checkpoints/{股票代码}_{YYYYMMDD}"
mkdir -p "$CHECKPOINT_DIR"

# 写入步骤结果
cat > "$CHECKPOINT_DIR/step{N}_{步骤名}.json" << 'JSONEOF'
{
  "step": N,
  "step_name": "{步骤名}",
  "timestamp": "ISO时间",
  "scores": { "子项1": 分数, "子项2": 分数, "total": 总分 },
  "key_findings": ["发现1", "发现2"],
  "risk_flags": ["风险1"],
  "next_step_inputs": { "供下一步使用的数据" }
}
JSONEOF

# 更新检查点状态
echo '{"last_step": N, "last_updated": "ISO时间", "status": "in_progress"}' > "$CHECKPOINT_DIR/checkpoint.json"
```

**⚠️ 检查点必须保存的场景**：
- Step 0 完成 → `step0_data.json`（含 industry_type 判定结果）
- Step 1 完成 → `step1_quality.json`（四维评分+依据）
- Step 2 完成 → `step2_financial.json`（四维评分+依据）
- Step 4 完成 → `step4_valuation.json`（估值区间+安全边际+使用的方法）
- Step 7 完成 → `step7_conclusion.json`（加权总分+投资结论+建仓建议）

### 0.2 多信源数据校验

**必校验10项关键字段**：当前股价&市值（≥3源）、PE-TTM、营收、净利润&增速、毛利率/净利率/ROE、分红&股息率、资产负债率&有息负债率、经营现金流&FCF、股东结构、行业地位/市占率。

**信源等级体系（六级视觉标注，强制执行）**：

| 视觉等级 | 名称 | 含义 | 典型来源 |
|:---:|---|---|---|
| 🟢🟢🟢🟢 | Tier 0 / S级API | 机器可读精确值 | yfinance / AkShare / jw-investment-data API / 交易所官方API |
| 🟢🟢🟢 | Tier 1 / 一级官方 | 公司公告/年报 | 公司年报、交易所披露(HKEX/SEC)、政府监管、审计师签字财报 |
| 🟢🟢 | Tier 2 / 二级权威 | 专业数据库 | Bloomberg/Reuters/FT/WSJ/财新、理杏仁/Wind/Choice、券商深度研报 |
| 🟢 | Tier 3 / 三级参考 | 垂直媒体 | 36氪/搜狐/东方财富/车家号、百度百科官方条目 |
| 🟡 | Tier 4 / 单源弱信源 | 仅1源或二级互抄 | 仅有一个信源的数据、二级媒体间"互相抄录"未追溯原始源 |
| 🔴 | Tier 5 / 传闻 | 仅小道消息 | 股吧评论、知乎匿名、供应链传闻、未证实爆料 |

**采信规则**：
- **价格类数据**：1个 🟢🟢🟢🟢（API）即可视为已满足"3源"基本要求；建议补1-2个 🟢🟢 网页源做交叉
- **财务指标**：至少2源，至少1个 🟢🟢 级以上
- **经营数据**（交付量/产能/市占率）：至少2源，至少1个 🟢🟢 级以上
- **若仅有API单源**：必须在标注栏显式写 `(xx API 单源 / 暂无网页交叉)`

**定量数据强制标注格式**（报告中每次引用定量数据时必须使用）：

```
格式一（行内括号，适合正文单点引用）：
2025年全年营收 **50亿元** 🟢🟢（来源：2025年报 + 财新报道）

格式二（表格相邻列，适合数据矩阵，推荐）：
| 指标 | 数值 | 信源等级 | 信源明细 |
|---|:---:|:---:|---|
| 2025营收 | 50亿 | 🟢🟢🟢 | 2025年报（一级官方） |
| 行业市占率 | 15% | 🟡 | 仅东方财富单源（二级互抄，无协会原始数据） |
```

**弱信源专项披露（强制）**：报告必须在"完整信源索引"章节设立 **"K.4 单源/弱信源清单"**，明列所有 🟡/🔴 级数据及其影响评估：

```markdown
### K.4 单源/弱信源清单（需读者特别警惕）

| 数据 | 等级 | 唯一/弱源 | 影响评估 | 后续验证建议 |
|---|:---:|---|---|---|
| [数据1] | 🟡 | [弱源说明] | [对报告结论的潜在影响] | [下次验证节点] |
```

若确实无弱源，写"经核验，无 🟡/🔴 级数据"。

**数据类型混淆防护（价格类数据专项）**：

> **血泪教训**：海尔 06690 v1.2 报告把研报目标价平均值 HK$31.30 误当"现价"，实际现价仅 20.86 HKD，价格高估30%，整份报告结论反转。**API 返回 `regularMarketPrice` 字段，从字段名层面就杜绝了混淆。**

**价格类数据的六种身份必须严格区分**：

| # | 数据身份 | 定义 | ⚠️ 混淆风险 |
|---|---------|------|-----------|
| 1 | **实时现价/最新收盘价** | 刚刚/当日成交价 | ✅ 这才是报告应采用的"当前股价" |
| 2 | **研报目标价(Target Price)** | 券商未来12个月预测价 | 🔴 绝不可作为现价 |
| 3 | **分析师一致预期价** | 多家券商目标价加权平均 | 🔴 同上 |
| 4 | **52周高/低/区间** | 过去52周历史极值 | 🔴 历史数据，不能当现价 |
| 5 | **发行价/IPO价/历史某时点价** | 历史成交价 | 🔴 与现价无直接关系 |
| 6 | **公允价值/DCF估值/内在价值** | 分析模型输出 | 🔴 是"值多少"不是"卖多少" |

**反混淆校验清单**（取数时必过5问）：
1. ✅ 这个数字是"当前成交价"吗？还是研报目标价/模型估值/历史数据？
2. ✅ 取数时间戳是几月几日几点？是否在最近3个交易日内？
3. ✅ 币种标清楚了吗？港股用港元、A股用人民币、ADR用美元？
4. ✅ 我是否从**至少3个独立权威源**看到了相同/高度接近的数字（差异≤2%）？
5. ✅ 如果是跨市场标的（AH/ADR），我是否对A股、H股、美股**分别独立取数**而非用汇率换算？

**任一问答"否"或"不确定"，立即停止，重新取数。**

**数据冲突处理**：差异≤5%取均值；5%-15%引入第三信源；>15%追溯公司公告；>30%暂停分析。

### 0.3 近30天重大事件扫描

**芒格三选一前置筛选**：可以投资（继续） / 不可以投资（排除） / 太难理解（放弃，90-95%归此类）

**必扫描5类事件**：监管/政策变动、管理层变动、重大战略发布、重大诉讼/安全事件、重大合作/并购/融资。同步必扫竞争对手动作。

**影响力分级**：🔴重大（影响>10%营收或触发一票否决）→ 报告首页单列；🟡显著（影响单一产品线）→ 对应模块引用调分；🟢一般 → 仅信源索引记录。

**灰犀牛扫描**：详见 Step 0.6 宏观环境评估中的灰犀牛五阶段和A股灰犀牛清单。

### 0.4 信息完整性检查

确认已收集：基本信息、股价数据（≥3源）、3-5年财务报表、分红数据、股东数据、管理层数据、行业数据、近期事件。关键信息缺失→补充取数后再进入 Step 1。

**审计报告四类意见与风险层级**（张新民）：无保留意见（风险最小）→ 保留意见（重点分析保留事项）→ 无法表示意见/否定意见（直接排除）。**关键审计事项**=企业最容易出问题的风险区域；收入确认/商誉减值/销售费用完整性被列为关键审计事项→利润操纵风险高。

### 0.4.1 行业类型判定（统一入口，后续所有Step引用）

> **⚠️ 强制前置**：在进入 Step 1 前，根据 Step 0 收集的行业数据，判定目标公司的行业类型。后续所有 Step（尤其是 Step 1.1 行业校准、Step 2 行业标准、Step 4 估值方法选择、Step 7 评分校准）均引用此处判定结果，**禁止跨Step重复判定**。

**判定流程**：

1. **查行业分类**：从 Step 0 comprehensive→plate 获取申万一级行业（如"保险"、"银行"、"食品饮料"、"半导体"）
2. **匹配下表**：按申万行业映射到行业类型，取对应的"核心特征"做确认
3. **输出 `industry_type`**：写入 step0_data.json 的 `industry_type` 字段（如 `"financial_insurance"`），后续所有Step直接引用此字段

| 行业类型 | 判定条件（匹配任一即可） | 核心特征 | 后续引用 |
|---------|----------------------|---------|---------|
| **金融-银行** | 申万"银行" | 资产驱动、息差为核心、PB估值 | Step 2(特殊标准), Step 4(PB/DDM), 7.1(行业校准) |
| **金融-保险** | 申万"保险" | 负债驱动、投资收益+承保利润、PEV估值 | Step 2(特殊标准), Step 4(PEV/PB), 7.1(行业校准) |
| **金融-券商** | 申万"非银金融"含券商 | 周期性强、β属性、PB+量能估值 | Step 4(PB+量能), 7.1(行业校准) |
| **金融-地产** | 申万"房地产" | 高杠杆、NAV估值、政策敏感 | Step 2(高负债率标准), Step 4(NAV/PB), 7.1(行业校准) |
| **周期-资源** | 申万"煤炭/钢铁/有色金属/石油石化" | 价格驱动、供需周期、产能重置成本 | Step 2(正常化利润), Step 4(正常化PE/产能市值比), 7.1(周期位置) |
| **消费-品牌** | 申万"食品饮料/家用电器/纺织服饰/美容护理" | 品牌溢价、渠道为王、DCF稳定 | Step 1.1(提价权), Step 4(DCF/PE分位) |
| **科技-成长** | 申万"电子/计算机/通信/传媒/半导体" | 高增长、研发驱动、PS/PEG估值 | Step 1.4(增长质量), Step 4(PEG/PS/DCF) |
| **医药-创新** | 申万"医药生物"含创新药 | 研发管线价值、DCF+管线估值 | Step 1.4(管线进展), Step 4(DCF+管线) |
| **制造-传统** | 申万"机械设备/电力设备/汽车/化工"等 | 规模经济、成本竞争、产能利用率 | Step 1.2(成本护城河), Step 4(DCF/PE分位) |
| **公用事业** | 申万"公用事业" | 稳定现金流、政策定价、股息率估值 | Step 4(DDM/股息率), 7.1(稳定性溢价) |

**判定输出格式**（写入 step0_data.json）：
```json
{
  "industry_type": "financial_insurance",
  "industry_sw_l1": "保险",
  "industry_calibration_key": "insurance",
  "valuation_method_override": ["PEV", "PB", "DDM"],
  "financial_standard_override": true
}
```

**⚠️ 边界情况处理**：
- 多元业务（如"金融+科技"）→ 按营收占比>60%的主营业务判定；若无单一>60%，取最高占比行业，但在 Step 4 同时用两种方法估值
- 行业转型期（如传统制造→新能源）→ 按新业务方向判定，但在 Step 1.1 标注转型风险
- 判定结果 **全局唯一**，后续 Step 禁止重新判定，仅引用此字段

### 0.5 公司背景研究（v3.61.2 增强版）

> 公司所处发展阶段直接影响后续估值方法选择、增长预期和风险判断。了解公司"从哪来"，才能判断"往哪去"。

**① 公司基本信息**（必填）：

| 信息项 | 内容 | 数据来源 |
|--------|------|---------|
| 公司全称 | — | 公司公告/年报 |
| 股票代码 | — | 交易所 |
| 上市时间 | — | 公司公告 |
| 所属行业 | 申万一级/二级 | 综合财务画像 |
| 注册地 | — | 公司公告 |
| 主营业务 | 核心产品/服务 | 年报 |
| 收入构成 | 各业务占比 | 年报/季报 |

**② 股权结构分析**（必填，v3.61.2 新增）：

> 股权结构决定了公司控制权归属、利益分配和治理质量。

**股权结构表**：

| 股东名称 | 持股比例 | 股东性质 | 一致行动人 | 质押情况 |
|---------|---------|---------|-----------|---------|
| 控股股东 | X% | 国有/民营/外资 | 是/否 | 质押Y% |
| 第二大股东 | X% | — | — | — |
| 第三大股东 | X% | — | — | — |
| ... | ... | ... | ... | ... |
| 合计（前十大） | X% | — | — | — |

**股权结构分析要点**：
1. **实际控制人**：最终控制方是谁？（个人/家族/国资委/无实控人）
2. **控股股东**：第一大股东持股比例？股东性质？（国有=治理规范但效率可能低；民营=灵活但可能有利益输送）
3. **股权集中度**：CR3>50%=高度集中（决策快但可能损害小股东）；CR3<30%=分散（可能有控制权争夺）
4. **一致行动人**：是否存在？影响实际控制权判断
5. **大股东质押率**：<30%安全 / 30-50%关注 / 50-80%风险高 / >80%直接排除

**股权结构风险信号**：
- 🔴 大股东质押率>80%
- 🔴 实际控制人被调查/立案
- 🔴 频繁关联交易+价格偏离市场
- 🟡 股权过于分散（无实控人）
- 🟡 大股东频繁减持

**③ 公司发展历史**（必填，v3.61.2 新增）：

> 发展历史揭示了公司的基因、能力和潜在风险。必须按时间线整理，识别关键转折点。

**发展历史时间线模板**：

| 时间 | 事件类型 | 事件描述 | 影响评估 | 成功/失败 |
|------|---------|---------|---------|----------|
| 创立年份 | 创立 | 公司成立/前身 | 奠定基础 | — |
| IPO年份 | 上市 | 交易所上市 | 融资扩张 | — |
| 关键年份1 | 转型/危机/突破 | 具体事件 | 改变命运 | 成功/失败 |
| 关键年份2 | 并购/剥离 | 具体交易 | 扩张/收缩 | 整合效果 |
| 关键年份3 | 战略调整 | 具体决策 | 方向变化 | 待观察 |
| 近期 | 最新事件 | 具体事件 | 未来方向 | 待观察 |

**事件类型分类**：
- **创立/上市**：公司诞生和资本化
- **重大转型**：主营业务变更/战略方向调整
- **重大危机**：经营困难/财务危机/法律纠纷
- **重大突破**：技术突破/市场突破/产品突破
- **重大并购**：收购标的/整合效果
- **重大剥离**：出售资产/业务重组
- **管理层变动**：核心高管更换
- **战略调整**：新战略/新方向

**发展历史分析要点**：
1. **公司是怎么发展到今天的？**
   - 内生增长型：靠自身产品/技术/渠道成长（如茅台）
   - 并购扩张型：靠收购扩张（如美的）
   - 政策驱动型：靠政策红利（如新能源）
   - 技术突破型：靠技术创新（如宁德时代）

2. **经历过哪些重大转折？**
   - 转型是否成功？（看转型后3年ROE变化）
   - 危机如何度过？（考验管理层应变能力）
   - 并购是否成功？（看整合后3年ROE变化）

3. **管理层决策风格的历史模式是什么？**
   - 激进型：高杠杆、频繁并购、追逐风口
   - 稳健型：适度负债、聚焦主业、长期主义
   - 保守型：低杠杆、现金充裕、增长缓慢

4. **企业文化从哪来？**
   - 创始人基因：创始人的背景和理念
   - 行业特性：行业惯例和竞争环境
   - 国企文化：政策导向、治理规范
   - 民企文化：灵活高效、风险偏好

**④ 发展阶段判断**：

| 阶段 | 特征 | 估值偏好 | 风险重点 |
|------|------|---------|---------|
| 初创期 | 收入小/亏损/高投入 | PS/用户价值 | 现金流断裂/产品失败 |
| 扩张期 | 收入高增/利润波动 | PEG/PS | 增长失速/资金链 |
| 成熟期 | 收入稳定/利润可观 | PE/DCF/股息率 | 护城河被侵蚀/管理层惰怠 |
| 转型期 | 收入结构调整/利润承压 | 分部估值 | 新业务不及预期/旧业务失速 |

**判断方法**：结合收入增速（>30%扩张/10-30%成熟/<10%衰退）、资本开支趋势（扩张期高/成熟期稳/衰退期低）、业务结构变化（新旧业务收入占比变化）综合判断。

**⑤ 企业基因判断**（历史视角，与Step 1.3管理层评估互补）：
- **本节（0.5）看"基因"**：公司是怎么发展到今天的？管理层决策风格的历史模式是什么？企业文化从哪来？
- **Step 1.3看"能力"**：当前管理层的资本配置、治理结构、战略执行力如何？未来能否持续？
- 简言之：0.5看过去（基因），1.3看现在和未来（能力）

**⑥ 近期事件**（近30天）：
- 重大公告（年报/季报/分红/增发/并购）
- 重大新闻（行业政策/公司事件/管理层变动）
- 重大研报（券商评级/目标价调整）

### 降级标记规范

- `[🔴矛盾]`：多源数据存在冲突

### 数据缺失处理

1. 该数据项标注 `⚠️数据缺失`
2. 对应评分项给予保守分数（取区间下限）
3. 在能力边界声明中说明缺失原因
4. 在 K.4 弱信源清单中记录
