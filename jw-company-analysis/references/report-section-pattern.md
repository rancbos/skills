# 新增报告板块的四步标准流程

> 当用户要求"在XX板块增加YY分析"时，按此流程执行。

## 流程

### Step 1: 更新模板（templates/company-analysis-report.md）
- 确定新板块在报告中的位置（哪个章节之后）
- 设计图表化输出格式（ASCII仪表盘/矩阵图/区间图/雷达图）
- 添加必要的表格模板

### Step 2: 更新 SKILL.md 执行骨架
- 在对应的 Step 中添加输出要求（标记为"⚠️ 强制"）
- 明确必须输出的内容清单
- 添加图表格式示例

### Step 3: 更新参考文档（references/）
- 如果涉及数据获取 → 更新 `references/data-preparation.md`
- 如果涉及分析方法 → 更新对应的 references 文件
- 确保 references 中的输出格式与模板一致

### Step 4: 一致性验证
- 运行 `bash scripts/audit_consistency.sh`
- 验证版本号一致（SKILL.md frontmatter + 标题）
- 验证模板中的图表格式与 SKILL.md 中的示例一致

## 图表化输出规范（用户偏好）

用户强烈偏好图表化输出，以下场景必须用图表：
- 关键财务指标 → 仪表盘/卡片图
- 事件扫描 → 时间线+影响力矩阵
- 周期定位 → 仪表盘/雷达图
- 产业链 → 流程图
- 估值区间 → 区间图
- 技术面 → 布林带图
- 评分 → 雷达图

格式要求：
1. ASCII 字符绘制（兼容纯文本）
2. 关键数据用框线突出
3. 颜色用 emoji（🟢好/🟡一般/🔴差）
4. 图表下方必须有简要说明

## Pitfall

- 模板和 SKILL.md 必须同步更新，否则报告会缺少新板块
- data-preparation.md 中的输出格式必须与模板一致（如 JSON 字段名）
- 图表不要太复杂，3-5个关键数据点即可，否则反而不便阅读
