# 禁区词速查与扩写防护

扩写内容时，最容易引入禁区词。本文档提供速查表和防护流程。

## 禁区词速查表

最容易在新内容中出现的禁区词：

| 禁区词 | 常见误用场景 | 替代写法 |
|--------|------------|---------|
| 这意味着 | 解释技术含义时 | 从此…/…从此…/ 直接删掉 |
| 本质上 | 做本质判断时 | 删掉，直接说判断 |
| 最后 | 列举的最后一项时 | 最终的/末端的/缺失的那一块 |
| 首先 | 对比中排第一时 | 骨子里/排在首位的是 |
| 其次 | 列举第二项时 | 紧随其后/另一维度 |
| AI工具 | 泛指AI产品时 | 说具体名字 |
| 某个模型 | 泛指模型时 | 说具体模型名 |
| 综上所述 | 总结段落开头 | 直接说结论 |
| 值得注意的是 | 引出重点时 | 删掉，直接说 |
| 不难发现 | 引出推论时 | 删掉，直接说 |
| 说白了 | 口语化解释时 | 换句话说/直白讲 |
| 换句话说 | 解释同一个意思 | 删掉，换个句式重写 |
| 不可否认 | 引出公认事实时 | 删掉 |
| 赋能 | 描述能力赋予 | 提供/增强/支撑 |
| 抓手 | 描述切入点 | 突破口/核心手段 |
| 打造闭环 | 描述完整流程 | 形成完整链路/实现端到端 |

## 扩写禁区词防护流程

### 场景

初稿质检通过（或修完禁区词后），因字数不够需要扩写。扩写时随手用了"本质上"、"这意味着"、"最后"等禁区词，导致下一轮质检又失败。

### 根因

扩写时注意力在内容深度上，不在禁区词上。新写的内容没有经过禁区词扫描就直接进入下一轮质检。

### 正确流程（节省3-5轮返工）

1. **扩写前**：心里默念禁区词清单
2. **扩写后**：立刻用 `search_files` 对新增段落做禁区词扫描
3. **修完再跑** `quality_check.py` 做最终确认

### 错误做法

扩写 → 直接跑质检 → 发现禁区词 → 修 → 再扩写 → 又发现禁区词 → 循环3-5次

这个循环每次浪费2-3分钟，累计可达10分钟。手动扫描一次只需要30秒。
