# 触发边界与 Skill 间关系

## 本 Skill 的定位

jw-hv-analysis 是**系统性深度研究**工具，核心特征：
- 双轴分析（纵向时间线 + 横向竞品对比）
- 信息密度高，需要联网搜索
- 最终产出 10,000-25,000 字的研究报告
- 有故事弧线和深度洞察，不是信息罗列

## 触发条件

### 应该触发

| 用户表达 | 判断依据 |
|---------|---------|
| "帮我用横纵分析法研究XX" | 明确指定方法论 |
| "帮我做个深度研究" | 强调深度、系统性 |
| "帮我摸清楚XX是怎么回事" | 需要全面了解 |
| "帮我搞懂这个东西" | 需要系统性理解 |
| "帮我做个deep research" | 英文表达同义 |
| "XX是什么来头" + 上下文暗示需要深度 | 需要判断上下文 |
| "帮我研究一下XX" + 竞品对比需求 | 需要横向分析 |

### 不应该触发

| 用户表达 | 应使用 |
|---------|--------|
| "XX是什么" | 通用对话，直接解释 |
| "帮我总结一下XX" | jw-content-summary |
| "帮我分析一下这家公司值不值得投" | jw-company-analysis |
| "帮我写篇公众号文章" | 通用对话或专门写作 skill |
| "帮我做个标题摘要" | 通用对话 |
| "XX和YY哪个好"（简单对比） | 通用对话 |
| "帮我看看这个代码" | 通用对话或代码相关 skill |

## 与相似 Skill 的区别

### jw-hv-analysis vs jw-company-analysis

| 维度 | jw-hv-analysis | jw-company-analysis |
|------|----------------|---------------------|
| 研究对象 | 产品、公司、概念、人物、技术 | 仅限公司 |
| 分析框架 | 双轴分析（时间+竞品） | 价值投资框架（财务+护城河+估值） |
| 最终产出 | 深度研究报告 | 投资分析报告 |
| 核心关注 | 发展历程、竞争格局、洞察 | 财务指标、投资价值、风险 |
| 触发场景 | "帮我研究XX" | "这家公司值不值得投" |

**重叠场景**：研究一家公司时，两个 skill 都可能触发。
**判断规则**：用户关注投资价值 → jw-company-analysis；用户关注发展历程和竞争格局 → jw-hv-analysis。

### jw-hv-analysis vs jw-content-summary

| 维度 | jw-hv-analysis | jw-content-summary |
|------|----------------|---------------------|
| 研究深度 | 深度研究，需要联网 | 快速蒸馏，基于已有内容 |
| 信息来源 | 联网搜索为主 | 用户提供的文档/书籍 |
| 输出长度 | 10,000-25,000字 | 2,000-5,000字 |
| 核心价值 | 独立研究和洞察 | 提炼核心方法论 |
| 触发场景 | "帮我深度研究XX" | "帮我总结这本书/这篇文章" |

**重叠场景**：研究一本书或一个方法论时。
**判断规则**：用户给了一本书/文章要总结 → jw-content-summary；用户要研究一个概念/产品的全貌 → jw-hv-analysis。

### jw-hv-analysis vs 公众号写作

| 维度 | jw-hv-analysis | 公众号写作 |
|------|----------------|----------------|
| 产出形式 | 研究报告（PDF/飞书） | 公众号文章 |
| 写作风格 | 研究报告 + 聊天感 | 卡兹克文风（口语化、去小标题） |
| 信息密度 | 高，需要大量引用 | 中，注重可读性 |
| 触发场景 | "帮我做个深度研究" | "帮我写篇公众号文章" |

**重叠场景**：用户想写一篇深度分析文章。
**判断规则**：最终要发公众号 → 通用对话或专门写作 skill；需要系统性研究报告 → jw-hv-analysis。

## 灰色地带处理

当用户表达模糊时，按以下优先级判断：

1. **有明确方法论指定**（如"用横纵分析法"）→ jw-hv-analysis
2. **有明确产出形式指定**（如"写篇公众号"）→ 通用对话或专门写作 skill
3. **有明确分析角度指定**（如"值不值得投"）→ 对应 skill
4. **以上都没有** → 根据上下文判断：
   - 用户是投资者/关注财务 → jw-company-analysis
   - 用户有具体文档要总结 → jw-content-summary
   - 其他 → jw-hv-analysis（默认深度研究）

## 多 Skill 协作

某些场景需要多个 skill 协作：

1. **先深度研究，再写公众号**：jw-hv-analysis 产出研究报告 → 专门写作 skill 基于报告写文章
2. **先研究公司，再做投资分析**：jw-hv-analysis 产出发展历程 → jw-company-analysis 补充财务分析
3. **先总结内容，再深度研究**：jw-content-summary 蒸馏核心概念 → jw-hv-analysis 扩展研究

这种情况下，明确告知用户各 skill 的分工，按顺序执行。
