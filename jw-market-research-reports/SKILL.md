# 市场研究报告

## 概述

市场研究报告是全面的战略文档，用于分析行业、市场和竞争格局，为商业决策、投资战略和战略规划提供依据。本技能支持**飞书云文档（1.5万字+）**和**PDF报告（50页+）**两种输出格式，风格对标麦肯锡、波士顿咨询、贝恩、高德纳和弗雷斯特等顶级咨询公司的交付物。

**核心特性：**
- **双输出格式**：支持LaTeX→PDF（50页+专业报告）和飞书云文档（1.5万字+结构化长文）
- **丰富的可视化内容**：在开始时生成5-6个关键图表（写作过程中按需增加更多）
- **多源数据收集**：`research-lookup` 优先，内置搜索工具（tavily/ddg）和金十数据作为备选
- **多框架方法**：波特五力模型、PESTLE、SWOT、波士顿矩阵、TAM/SAM/SOM
- **专业排版**：咨询公司级别的字体、配色和版式（LaTeX路径）/ 结构化富文本（飞书路径）
- **可操作的建议**：以战略为重点，附实施路线图

## 适用场景

本技能适用于以下场景：
- 为投资决策创建全面的市场分析
- 为战略规划编写行业报告
- 分析竞争格局和市场动态
- 进行市场规模评估（TAM/SAM/SOM）
- 评估市场进入机会
- 为并购活动准备尽职调查材料
- 为行业定位创建思想领导力内容
- 编写进入市场策略文档
- 分析监管和政策对市场的影响
- 为新产品发布构建商业案例

## 前置检查（每次执行前必须完成）

在开始报告生成之前，执行以下检查：

### 1. 依赖skill可用性检查

```bash
# 检查 scientific-schematics 是否已安装
ls ~/.hermes/skills/scientific-schematics/scripts/generate_schematic.py 2>/dev/null && echo "OK" || echo "MISSING"

# 检查 generate-image 是否已安装
ls ~/.hermes/skills/generate-image/scripts/generate_image.py 2>/dev/null && echo "OK" || echo "MISSING"

# 检查 research-lookup 是否已安装
ls ~/.hermes/skills/research-lookup/scripts/research_lookup.py 2>/dev/null && echo "OK" || echo "MISSING"
```

**处理策略：**
- `scientific-schematics` 缺失 → 可视化图表跳过，报告以纯文字+表格形式输出
- `generate-image` 缺失 → 信息图/封面图跳过
- `research-lookup` 缺失 → 使用内置搜索工具（`tavily_search`、`mcp_ddg_mcp_search`、`mcp_jin10_*`）替代
- 所有工具缺失 → 仍可生成纯文字分析报告

### 2. 输出格式选择

根据用户指定或默认策略选择输出格式：

| 条件 | 输出格式 | 篇幅指标 |
|------|---------|---------|
| 用户指定"飞书"/"云文档" | 飞书云文档 | 1.5万字+ |
| 用户指定"PDF"/"LaTeX" | LaTeX→PDF | 50页+ |
| 用户未指定（默认） | 飞书云文档 | 1.5万字+ |

### 3. 章节优先级调整

根据行业特征调整章节权重（不必机械执行11章固定结构）：

| 行业类型 | 核心章节（必须深入） | 可精简章节 |
|---------|-------------------|-----------|
| 资源/原材料型（铝、钢、锂） | 产业链分析、供给侧约束、成本结构 | 技术格局、客户分析 |
| 消费品型（白酒、食品、服装） | 品牌梯队、消费场景、渠道分析 | 技术格局、投资论证 |
| 周期型（基建、工程机械） | 周期位置、政策驱动、替代需求 | 客户分析、创新格局 |
| 科技型（AI、半导体） | 技术路线、专利格局、研发投入 | 监管环境、实施路线图 |

**调整原则：** 核心章节可扩展至8-10页，精简章节可压缩至1-2页或合并。

## 工作流程

### 阶段一：研究与数据收集

**步骤1：确定范围**
- 明确市场定义
- 设定地理边界
- 确定时间跨度
- 识别需要回答的关键问题

**步骤2：深入研究**

多源数据收集（按优先级使用）：

**方案A：使用 `research-lookup`（优先）**

```bash
# 市场规模和增长数据
python skills/research-lookup/scripts/research_lookup.py \
  "What is the current market size and projected growth rate for [MARKET] industry? Include TAM, SAM, SOM estimates and CAGR projections"

# 竞争格局
python skills/research-lookup/scripts/research_lookup.py \
  "Who are the top 10 competitors in the [MARKET] market? What is their market share and competitive positioning?"

# 行业趋势
python skills/research-lookup/scripts/research_lookup.py \
  "What are the major trends and growth drivers in the [MARKET] industry for 2024-2030?"

# 监管环境
python skills/research-lookup/scripts/research_lookup.py \
  "What are the key regulations and policy changes affecting the [MARKET] industry?"
```

**方案B：使用内置搜索工具（`research-lookup` 不可用时）**

```
# 市场规模 → tavily_search / mcp_ddg_mcp_search 搜索行业报告
# 竞争格局 → 搜索上市公司年报、券商研报
# 行业趋势 → mcp_jin10_list_flash / mcp_jin10_search_flash 获取实时行业快讯
# 监管环境 → 搜索政府网站、行业协会公告
# 财经数据 → mcp_jin10_get_quote 获取实时行情，mcp_jin10_list_calendar 获取财经日历
```

**方案C：中文行业专用数据源（A股/中国市场）**

```
- 国家统计局（data.stats.gov.cn）：GDP、工业增加值、产能利用率
- 海关总署：进出口数据
- 行业协会：有色金属工业协会、工程机械工业协会等
```

**步骤3：数据整理**
- 创建 `sources/` 文件夹存放研究笔记
- 按章节整理数据
- 识别数据缺口
- 按需进行补充研究

**⚠️ 阶段一质量门（进入阶段二前必须通过）：**

以下条件全部满足方可进入阶段二：
- [ ] 至少找到当前市场规模数据（1个来源）
- [ ] 至少找到3家主要竞争对手的基本信息
- [ ] 至少找到1个增长率/趋势数据点
- [ ] 已识别主要数据缺口清单

**研究深度控制（防止过度研究）：**
- **核心章节**（市场规模、竞争格局、风险分析）：最多搜索3轮，每轮3-5个查询
- **标准章节**（驱动因素、监管环境）：最多搜索2轮
- **精简章节**（技术格局、客户分析，如已判定为可精简）：最多搜索1轮
- **停止研究的信号**：连续2次搜索返回重复信息，或已覆盖80%的章节数据需求

### 阶段二：分析与框架应用

**步骤3.5：选择适用分析框架（必须在应用框架前完成）**

根据行业特征选择适用框架，不必全部强制应用：

**必选框架（所有行业通用）：**
- **市场规模评估**（TAM/SAM/SOM）：即使是成熟行业，也需要量化市场边界
- **竞争格局分析**：市场集中度、主要玩家、竞争强度
- **驱动因素与趋势**：宏观+行业层面的增长/抑制因素
- **风险分析**：概率×影响矩阵

**选配框架（按行业类型选择）：**

| 框架 | 适用场景 | 不适用场景 |
|------|---------|-----------|
| 波特五力 | 行业结构分析（几乎所有行业） | 已有最新五力分析可直接引用 |
| PESTLE | 政策敏感型行业（能源、医疗、教育） | 纯技术驱动型行业 |
| SWOT | 企业级分析 | 行业级分析（SWOT更适合单个公司） |
| 波士顿矩阵 | 多产品线企业战略 | 单一行业/单一产品分析 |
| 客户旅程 | B2C消费品、SaaS | B2B原材料、大宗商品 |
| 技术路线图 | 科技、医药、新能源 | 传统行业（白酒、有色金属） |
| 战略群组 | 竞争者众多且差异化明显 | 寡头垄断市场 |

**原则：** 选配框架如不适用，直接跳过，不要为凑框架而生搬硬套。

**步骤4：应用分析框架**

对每个框架进行结构化分析：

- **市场规模评估**：TAM → SAM → SOM，明确假设
- **波特五力模型**：对每个力量进行高/中/低评级并说明理由
- **PESTLE**：分析每个维度的趋势和影响
- **SWOT**：内部优势/劣势，外部机会/威胁
- **竞争定位**：定义坐标轴，标绘竞争对手

**步骤5：形成洞察**
- 将研究发现综合为关键洞察
- 识别战略启示
- 制定建议
- 对机会进行优先排序

**⚠️ 数据冲突处理规则：**

当多个来源数据不一致时：
1. **来源优先级**：政府统计 > 行业协会 > 上市公司年报 > 券商研报 > 媒体报道
2. **差异<10%**：取中位数，标注"综合多源数据"
3. **差异10%-30%**：取区间估计（如"4500-5500万吨"），标注各来源数值
4. **差异>30%**：显式标注矛盾，说明各来源口径差异原因，取最保守估计
5. **时效性原则**：优先采用最新数据，标注数据截止日期

**⚠️ 阶段二质量门（进入阶段三前必须通过）：**

- [ ] 核心框架（市场规模、竞争格局、风险）分析完成
- [ ] 每个分析结论有≥1个数据支撑
- [ ] 已识别3-5个关键洞察（不是数据罗列，是"所以呢"的判断）

**🔄 迭代回退机制：**

如果阶段四撰写时发现某章节数据严重不足：
1. 暂停撰写，回到阶段一补充搜索（限1-2轮）
2. 补充完成后直接回到阶段四继续撰写
3. 不需要重新执行阶段二/三
4. 如补充搜索仍无结果，使用 `[待补充：XXX]` 标注并继续

### 结构完整性
- [ ] 执行摘要（关键发现 + 投资论证 + 战略建议）
- [ ] 所有核心章节均已包含（11章标准结构或按行业调整后的结构）
- [ ] 附录：研究方法、数据表、企业概况
- [ ] 所有数据点均有来源

### 可视化完整性（如工具可用）
- [ ] 市场增长轨迹图（优先级1）
- [ ] TAM/SAM/SOM图（优先级2）
- [ ] 波特五力模型（优先级3）
- [ ] 竞争定位矩阵（优先级4）
- [ ] 风险热力图（优先级5）

### 内容质量
- [ ] 所有统计数据均有来源
- [ ] 预测包含假设条件
- [ ] 框架应用得当
- [ ] 建议可操作
- [ ] 写作质量专业
- [ ] 无占位符或未完成的章节

### 输出质量
- [ ] 飞书文档：链接可正常打开（www.feishu.cn 域名）
- [ ] 飞书文档：权限为 anyone_readable
- [ ] 飞书文档：总字数 ≥1.5万字
- [ ] PDF：编译无错误、页数 ≥50页
- [ ] 可视化图表正确渲染（如有）

## 常见问题排查

**问题**：飞书文档链接打不开
- **解决方案**：执行 `permission.public patch` 改为 `anyone_readable`，用 `drive +inspect` 获取 `www.feishu.cn` 域名URL，直接在消息文本中贴链接（不走消息卡片）

**问题**：可视化图表未生成
- **解决方案**：检查 `scientific-schematics` 和 `generate-image` 是否已安装，检查 `OPENROUTER_API_KEY` 是否配置。不可用时采用纯文字+表格模式

**问题**：报告篇幅不足
- **解决方案**：扩展附录中的数据表，增加更详细的公司概况，包含额外的区域分解。飞书版目标1.5万字+，PDF版目标50页+

**问题**：参考文献缺失条目
- **解决方案**：在首次xelatex编译后运行bibtex，检查.bib文件是否有语法错误

**问题**：表格/图表溢出（LaTeX路径）
- **解决方案**：使用 `\resizebox` 或 `adjustbox` 宏包，减小图片宽度百分比

**问题**：生成的可视化质量不佳
- **解决方案**：使用 `--doc-type report` 标志，使用 `--iterations 5` 增加迭代次数
