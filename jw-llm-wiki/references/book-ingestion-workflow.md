# 书籍批量入库工作流

## 背景

当用户要求将 /root/books/ 下的书籍存入知识库时，按此流程执行。

## 前置检查

1. 找到书籍文件：`ls /root/books/ | grep -i "关键词"`
2. 检查文件大小：`wc -l <file>` — 超过 1000 行走完整处理流程
3. 隐私自查（用户回复 y 后继续）

## 执行步骤

```
1. cp 书籍文件 → /root/wiki-ai/raw/articles/{日期}-{书名}.md
2. cache.sh check（确认 MISS:no_entry）
3. 读取前 500 行了解结构
4. 生成素材摘要页 → wiki/sources/{日期}-{书名}.md
5. 提取关键概念，逐个创建实体页 → wiki/entities/{概念}.md
6. 创建主题页 → wiki/topics/{主题}.md
7. 更新 index.md（全量重写）
8. 更新 log.md（追加记录）
9. cache.sh update（更新缓存）
```

## 多本书同时入库

用户说"把 A 和 B 两本书存入知识库"时：
- 两本书各自独立走完整流程
- 共享实体页（如两本书都提到"迈克尔·波特"，实体页 sources 数组合并）
- 主题页可以合并（如"竞争优势"和"竞争战略"都属于"战略管理"主题）
- index.md 最后一次性全量更新

## 实体页去重规则

- 同一个概念（如"五力模型"）只创建一个实体页
- 多本书引用同一概念时，在 sources 数组中添加多个来源
- "不同素材中的观点"章节分别列出每本书的描述

## 书籍文件名格式

统一用：`{YYYY-MM-DD}-{书名}.md` 或 `.txt`

原始文件扩展名保持不变（.md / .txt / .pdf），不要转换格式。
