# A股每日大势研判

> **Iron Law**: 结论先行，数据说话。必须先写市场状态判断，再展开分析。信号冲突时说清楚冲突在哪里，不要强行归类。宁可低置信度，也不要伪确定。


## 0. 触发条件

当用户出现以下任一需求时，使用这个 skill：
- 让你研判今天/明天的 A股大盘
- 让你判断当前市场处于什么阶段
- 让你分析主线板块、资金流向、情绪周期或轮动结构
- 让你输出盘后复盘、次日观察点
- 让你分析北向资金、融资融券等资金动向

## 1. 目标

输出要回答四个问题：
1. 当前市场处于什么状态？
2. 今天是谁在主导市场？
3. 主线板块和轮动方向是什么？
4. 明天需要观察什么关键变量？

## 2. 适用范围

只适用于 A股市场。不适用于港股、美股、纯个股价值分析。

## 3. 依赖技能

**主数据源**：
- **jw-investment-data**（路径：`/root/.hermes/skills/jw-investment-data/SKILL.md`）：行情与结构数据底座
  - 指数行情：上证、深证、创业板、科创50、沪深300
  - K线数据：近10/20/60日K线，用于计算均线、MACD、RSI等技术指标
  - 涨跌家数、涨停/跌停统计
  - 成交额与量能变化
  - 融资融券余额变化
  - 北向资金净流入/流出
  - 板块涨幅排行
  - **关键脚本**：`fetch_market_data.py`（行情/K线）、`technical_indicators.py`（技术指标）、`fetch_macro.py`（宏观数据）

**辅助数据源**（主数据源不足时补充）：
- **jw-news**（路径：`/root/.hermes/skills/jw-news/SKILL.md`）：事件信息层（政策、新闻、财经日历）
- **financial-data-mcp**（路径：`/root/.hermes/skills/data-science/financial-data-mcp/SKILL.md`）：实时补充与校验层（金十快讯、实时行情）
- **web-access**：搜索东方财富、同花顺等财经网站（最后手段）

**数据获取优先级**：
```
1. jw-investment-data（首选，数据最全面）
2. jw-news + financial-data-mcp（事件/快讯补充）
3. web-access 搜索（降级方案）
```

**降级策略**：
- 如果 jw-investment-data 脚本报错（如语法错误），先尝试修复或用 `--force` 参数
- 如果 jw-investment-data 脚本超时（默认60秒），直接跳过进入辅助数据源，不要重试浪费时间
- 如果主数据源完全不可用，用辅助数据源补充，并在输出中标注"数据来源：辅助渠道，置信度降低"
- 如果所有数据源都失败，明确标注"数据不足，以下判断为低置信度"

**⚠️ 已知陷阱**：
1. **jw-investment-data 脚本超时**：`fetch_market_data.py` 串行调用8个引擎，每个超时5-10秒，总最坏耗时80秒。熔断器需3次失败才触发。遇到超时直接降级到 web search，不要重试。用 `timeout 25` 包裹调用。详见 jw-investment-data 的 `references/timeout-and-circuit-breaker.md`。
2. **指数数据获取**：`fetch_market_data.py` 不支持指数代码（000001=平安银行≠上证指数）。指数行情必须通过 `mcp_minimax_search_web_search` 或 `mcp_tavily_tavily_search` 搜索获取。详见 jw-investment-data 的 `references/index-data-pitfall.md`。
3. **web search 数据不一致**：不同财经网站的成交额、涨跌家数可能有微小差异（因为统计口径不同），取多个来源的中位数即可，不必追求精确。
4. **盘中 vs 盘后数据差异**：盘中搜索到的数据可能是午盘数据而非收盘数据，注意区分"截至午盘"和"截至收盘"的表述。
5. **⚠️ 必须先调脚本再用 web search**：步骤1的数据采集必须先尝试 `technical_indicators.py` 获取个股技术指标（用 venv python 调用），再用 web search 补充脚本不覆盖的数据（指数行情、涨跌家数、板块涨幅、资金流向）。不要跳过脚本直接全用 web search，那样会丢失结构化技术指标数据。
6. **technical_indicators.py 必须用 venv python**：系统无 pip，baostock 安装在 venv 中。调用方式：`~/.hermes/skills/jw-investment-data/.venv/bin/python ~/.hermes/skills/jw-investment-data/scripts/technical_indicators.py --symbol 600519 --format markdown`。用系统 python3 调用会报 `No module named 'baostock'`。

## 4. 研判流程（6步）

执行顺序固定，每一步的结论喂给下一步。步骤之间有依赖关系，不要跳步。

**场景区分**：
- **快速研判**（步骤1-6，精简输出）：核心层结论，3秒看完
- **深度复盘**（步骤1-6，完整输出）：完整层报告，含全部数据和分析

### 步骤1：数据采集

**数据来源优先级**：
1. **首选**：调用 jw-investment-data 脚本获取结构化数据
2. **补充**：调用 jw-news 获取新闻事件、financial-data-mcp 获取实时快讯
3. **降级**：web-access 搜索财经网站

**采集流程**：
```bash
# ⚠️ 必须用 python3（非 python），否则中文注释导致 SyntaxError
# ⚠️ 必须用 timeout 25 包裹，防止脚本超时阻塞（8引擎串行调用，最坏80秒）

# 1. 获取技术指标（先于 web search，用 venv python）
~/.hermes/skills/jw-investment-data/.venv/bin/python ~/.hermes/skills/jw-investment-data/scripts/technical_indicators.py --symbol 600519 --format markdown

# 2. 获取指数行情（脚本不支持指数，必须用 web search）
mcp_minimax_search_web_search(query="A股 今日行情 上证指数 深证成指")

# 3. 获取K线数据（可选，用于个股深度分析）
cd ~/.hermes/skills/jw-investment-data/scripts
timeout 25 python3 fetch_market_data.py --category kline --symbol 600519 --start 20260101 --format markdown

# 4. 获取宏观数据（如需要）
timeout 25 python3 fetch_macro.py --category all --format markdown
```

**⚠️ 指数数据获取**：
- `fetch_market_data.py` 的 7 个引擎均不支持指数查询（上证/深证/创业板/科创50/沪深300）
- 指数行情必须通过 `mcp_minimax_search_web_search` 或 `mcp_tavily_tavily_search` 搜索获取
- 详见 jw-investment-data 的 `references/index-data-pitfall.md`

**采集内容**（按优先级排序）：
- [ ] 主要指数表现（上证、深证、创业板、科创50、沪深300）
- [ ] 成交额与较前日变化（**相对值比绝对值更重要**）
- [ ] 近10个交易日的成交额序列
- [ ] 涨跌家数、涨停家数、跌停家数、炸板率
- [ ] 涨跌家数比例（如 1:2.5）
- [ ] 连板梯队分层（首板、2板、3板、高度板）
- [ ] 板块涨幅排行
- [ ] **融资融券余额变化**（杠杆资金动向）
- [ ] **北向资金净流入/流出**（外资动向）
- [ ] 重要新闻、政策、事件（通过 jw-news 获取）

**数据不足处理**：
- 如果 jw-investment-data 脚本报错，记录错误并尝试修复
- 如果主数据源缺失某项数据，用辅助数据源补充
- 最终输出时必须标注数据来源和完整度

### 步骤2：事件驱动判断

**事件分级**：

| 级别 | 举例 | 影响力 |
|------|------|--------|
| 重大 | 央行降息/降准、国务院级别政策、战争爆发、GDP大幅超/不及预期 | 可能改变市场趋势 |
| 中等 | 部委级政策、行业规划、重要公司财报、PMI/CPI偏离预期 | 影响相关板块 |
| 轻微 | 地方政策、个别公司事件、媒体吹风 | 有限影响，可能被忽略 |

**事件影响判断**：
```
如果 事件级别 >= 重大 且 市场未充分反应：
  → 可能触发趋势改变，市场格局面临重塑

如果 事件级别 = 中等 且 市场未充分反应：
  → 影响相关板块，调整板块配置

如果 事件 = 利好落地 且 市场已充分反应：
  → "买预期卖事实"，警惕冲高回落

如果 事件 = 利空落地 且 市场已充分反应：
  → "利空出尽"，可能触底反弹
```

**事件脱敏效应**：
- 同一类型事件多次发生后，市场反应会钝化
- 判断标准：比较该事件近期出现频率与历史均值——如果近30天出现次数 > 近6个月月均次数的2倍，视为已脱敏，影响力度降一级

### 步骤3：市场状态、情绪与风格判断（核心）

将市场状态、情绪周期、风格判断合并为一步，因为三者高度耦合——市场状态是情绪的外在表现，风格是情绪的传导机制。

**第一层：市场状态判断**

| 状态 | 核心特征 | 典型情绪阶段 |
|------|----------|-------------|
| 趋势市 | 指数、广度、量能三者共振，主线明确 | 高潮→延续 |
| 震荡市 | 指数区间波动，板块轮动频繁 | 修复/混沌 |
| 修复市 | 前期退潮后情绪回暖，但未确认趋势 | 冰点→修复 |
| 退潮市 | 涨停减少、跌停增加、资金流出 | 退潮→冰点 |
| 混沌市 | 没有明确主线，轮动快，强弱切换 | 混沌 |

**决策树（使用相对值，不依赖绝对阈值）**：

> **术语定义**：本文中"首板晋级率"= 当日首板个股中次日成功晋级二板的比例；"连板晋级率"= N板个股中次日晋级(N+1)板的比例。计算基数为当日实际封板数（不含炸板）。"炸板率"= 当日触及涨停但未封住（开板/回落）的个股数 ÷ 当日触及涨停总数 × 100%。

```
如果 涨停家数 > 20日均值的1.5倍 且 跌停 < 20日均值的0.5倍
   且 成交额 > 20日均值 且 较前日放量10%以上
   且 首板晋级率 > 30%：
  → 趋势市

如果 涨停家数在20日均值附近（0.7~1.3倍）
   且 跌停在20日均值附近
   且 成交额在20日均值附近
   且 涨跌家数比例在 1:1 ~ 1:2 之间：
  → 震荡市或修复市

如果 涨停家数 < 20日均值的0.5倍 且 跌停 > 20日均值的2倍
   且 成交额 < 20日均值的0.8倍
   且 涨跌家数比例 > 1:3：
  → 退潮市或混沌市
```

**模糊区间处理**：如果多个信号指向不同状态（如涨停多但跌停也在增加），不要强行归类，而是写明"当前处于 [状态A] 和 [状态B] 的模糊地带，信号冲突点在于 xxx"，置信度取较低值。

**优先级规则**：
- 如果多个信号冲突，以"涨停家数/跌停家数"比例为主
- 成交额的相对变化比绝对值更重要

**第二层：情绪周期细分**

| 阶段 | 关键信号 | 市场含义 |
|------|----------|----------|
| 冰点 | 跌停/亏钱效应显著，首板少，高度板断裂 | 市场极度悲观，卖盘枯竭 |
| 修复 | 情绪开始回升，首板增多，但连板晋级率低 | 抛压减轻，资金试探性进场 |
| 高潮 | 涨停多、连板高、情绪亢奋、首板晋级率高 | 市场亢奋，警惕情绪退潮 |
| 退潮 | 高度板断裂、炸板率高、亏钱效应扩散 | 获利盘出逃，市场分歧加大 |
| 混沌 | 无一致方向，强弱切换快，轮动无序 | 多空博弈激烈，方向不明 |

**情绪尾部特征**（退潮前兆，出现2个以上就要警惕，整体表明市场进入高风险阶段，赚钱效应快速消退，亏钱效应开始扩散）：
- 地天板/天天地频繁出现
- 龙头复盘后补涨股集体兑现
- 首板晋级二板概率持续 < 10%
- 高度板断裂（三进四没了）
- 周一情绪本应最好但仍走弱

**第三层：风格判断**

判断当前是量化主导还是游资主导：

| 维度 | 量化主导 | 游资主导 |
|------|----------|----------|
| 开盘表现 | 联动下跌，不给反弹 | 有犹豫、有试探、有从容卖点 |
| 首板晋级率 | < 10% | > 30% |
| 高度板 | 断裂（三进四没了） | 可突破八板 |
| 卖点 | 确认即走，不能等反弹 | 可等分时高点 |

**风格含义**：风格判断会影响对市场状态的解读：
- 量化主导：即使市场状态是"趋势市"，短线活跃度也受限，波动可能被算法平滑
- 游资主导：连板活跃，市场投机氛围浓厚

### 步骤4：资金面判断

**判断资金流向**：

| 资金状态 | 特征 | 市场含义 |
|----------|------|----------|
| 持续流入 | 成交额放大、热点集中、主线明确 | 市场做多意愿强烈，趋势延续 |
| 脉冲式流入 | 单日放量但不持续 | 资金分歧，持续性存疑 |
| 选择性流入 | 只有部分板块涨，其他不动 | 结构性行情，非普涨 |
| 退潮式流出 | 成交额萎缩、热点分散 | 市场信心不足，观望情绪浓 |
| 存量博弈 | 成交额稳定、板块轮动快 | 场内资金博弈，增量有限 |

**关键指标**（使用相对值）：
- 成交额较20日均值的变化比绝对值更重要
- 持续缩量后放量（较前日+20%以上）：可能是资金回流信号
- 持续放量后缩量（较前日-20%以上）：可能是资金撤离信号

> **"持续"定义**：连续2日以上同方向变化才算"持续"，单日波动视为噪音。

**外资动向判断标准**：
- 北向资金单日净流入 > 近20日均值的2倍 → 外资明显流入，市场有支撑
- 北向资金单日净流出 > 近20日均值的2倍 → 外资明显流出，承压较大
- 连续3天同方向流动（且单日金额不低于近20日均值的0.5倍）→ 趋势信号，比单日更有意义

**融资融券**：
- 融资余额持续增加 → 杠杆资金进场，市场偏多
- 融资余额持续减少 → 杠杆资金撤退，市场偏空
- 融券余额突增 → 做空力量增强，市场分歧加大

### 步骤5：板块轮动与技术面判断

**板块估值分位获取**：jw-investment-data 不支持板块级数据，需用 web search 漏斗式搜索获取。详见 jw-investment-data 的 `references/web-search-valuation-pattern.md`。

**板块生命周期**：

| 阶段 | 特征 | 市场含义 |
|------|------|----------|
| 预期酝酿 | 有催化但还没涨 | 市场关注度提升，资金观望 |
| 启动 | 首板出现、趋势突破 | 资金开始介入，板块升温 |
| 加速 | 连板增多、资金涌入 | 做多共识形成，赚钱效应强 |
| 高潮 | 涨停最多、情绪亢奋 | 板块过热，警惕分歧 |
| 退潮 | 高度板断裂、炸板增多 | 获利盘出逃，板块走弱 |
| 反抽 | 退潮后短暂反弹 | 资金尝试修复，持续性存疑 |
| 结束 | 没有新催化 | 资金撤离，板块进入调整 |

**板块锚点思维**（外部参考）：
- 美股核心标的（如英伟达、谷歌）对A股相关板块有定价指引：谷歌走势→AI应用方向，英伟达走势→算力方向
- 数据源：通过 `mcp_minimax_search_web_search` 搜索"英伟达 股价"或"谷歌 股价"获取最新走势

**轮动速度判断**：
- 轮动过快（板块/题材概念日内切换2次以上，以同花顺或东方财富板块分类为准）→ 混沌市特征，不宜追热点
- 轮动有序（主线板块持续3天以上）→ 趋势市特征，可以跟随主线

**技术面判断**（只看结构，不看单根K线）：

均线系统：
- 5日线：短线生命线，跌破表示短期走弱，站稳表示短期强势
- 10日线：短期趋势线，是短线回踩的常见支撑
- 20日线：中期趋势线，是波段方向的锚点
- 60日线：牛熊分界线，跌破意味着中期趋势转弱

结构定义：
- **上升结构**：高点抬高、低点抬高、站在5/10/20日线之上
- **下降结构**：高点降低、低点降低、被5/10/20日线压制
- **箱体结构**：在明确的上下沿之间震荡，均线缠绕

量价关系：
- 放量突破 → 有效突破概率高
- 缩量突破 → 假突破概率高
- 放量下跌 → 恐慌盘释放，可能是阶段性底部
- 缩量下跌 → 阴跌不止，底部未到

**九转序列使用规则**：
- 前提：先确定大趋势方向
- 主升正向波：高九容易失效（不必重视），低九容易失效（不必重视）
- 主升逆向波：低九可作为反弹信号，高九可作为回调信号
- 判断口诀：正向波的九转容易失效，逆向波的九转才有效

**MACD金叉**：
- 60分钟级别出现金叉+红角线 → 短期反弹信号
- 反弹周期通常在8-15个交易日，需结合量能确认持续性

### 步骤6：综合研判

**信号一致性判断**（综合步骤3-5的所有信号）：

```
信号一致（全部偏多/全部偏空）→ 市场方向明确，置信度高
信号冲突（部分偏多部分偏空）→ 市场分歧大，置信度降低
数据不足或判断模糊 → 市场处于混沌状态，需要进一步观察
```

**风格对市场的影响**：
- 量化主导时：即使信号偏多，短线波动也可能被算法平滑
- 退潮市+冰点：无论其他信号如何，市场都处于高风险状态
- 风格切换当天：市场处于不稳定状态，需要观察新风格能否确立

**一句话结论的写法**：
- 优先级：市场状态 > 主线方向 > 市场特征
- 格式：`市场处于 [状态] 阶段，主线是 [板块]，当前市场特征偏 [强势/震荡/弱势]，重点观察 [关键变量] 是否延续。`
- 不要用"关注""留意""等待"等空话，必须给出明确的市场判断

**分层输出**：

核心层（快速版，适合临时询问，3秒看完）：
```
【结论】市场处于 xxx 阶段，特征偏 xxx
【主线】xxx
```

完整层（深度复盘版）：📚详见 `references/output-template.md`

## 5. 质量控制

### 自检清单

输出前检查（每项对应步骤编号）：
- [ ] 是否写了市场状态？（→ 步骤3）
- [ ] 是否写了情绪周期阶段？（→ 步骤3）
- [ ] 是否写了风格判断（量化/游资）？（→ 步骤3）
- [ ] 是否写了资金面判断？（→ 步骤4）
- [ ] 是否写了主线板块？（→ 步骤5）
- [ ] 是否写了市场特征和风险提示？（→ 步骤6）
- [ ] 是否写了至少2条失效条件？（→ 步骤6）
- [ ] 是否写了次日观察点？（→ 步骤6）
- [ ] 是否避免了空话（"关注""留意""等待"）？（→ 全局）
- [ ] 如果信号冲突，是否明确写了冲突点和处理方式？（→ 步骤3）

### 失效与降级规则

如果出现以下情况，主动降级结论：
- 关键数据缺失
- 盘面与事件完全背离
- 市场风格切换过快，无法确认主线
- 指数与个股广度严重背离
- 信号冲突且无法判断优先级

降级时明确写：
- 目前只能给出低置信度判断
- 暂时不适合给出明确方向判断
- 需要等待下一交易日验证

### 置信度规则

| 置信度 | 条件 |
|--------|------|
| 高 | 指数结构清晰、广度和量能配合、主线明确、资金持续、事件与盘面一致 |
| 中 | 满足大部分条件，但有一到两个关键变量未确认 |
| 低 | 数据缺失、盘面冲突明显、主线不清、情绪极端但缺乏延续 |

## Anti-Patterns（强制阅读）

1. 只看指数，不看广度
2. 只看单日涨跌，不看结构
3. 把脉冲行情当趋势
4. 把情绪高潮当主升起点
5. 把事件利好等同于盘面利好
6. 把局部抱团误判为全面修复
7. 不写失效条件
8. 不写市场特征和风险提示
9. 用空话代替判断
10. 在量化主导环境下还用游资思维分析短线
11. 不区分主升正向波和逆向波就乱用九转序列
12. 忽略事件脱敏效应，对重复事件过度反应
13. 不写风格判断，不知道当前是量化还是游资主导
14. 信号冲突时强行归类，不写模糊地带

### 最终原则

- 先识别市场状态
- 再识别主导力量
- 再识别资金方向
- 再写清走势特征和失效条件

宁可保守，不要硬猜。
宁可少说，不要空说。
宁可低置信度，也不要伪确定。
信号冲突时说清楚冲突在哪里，不要装作一切清晰。
