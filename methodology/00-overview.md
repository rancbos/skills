# 书籍总结流水线总览

本文是 content-summary-jw 所用 SOP 的设计说明，解释"为什么这么做"。执行时的具体步骤请读 `SKILL.md` 和 `methodology/` 下的各阶段文件。

## 命名

**四阶段流水线** =
- **阶段 1**: 整书理解（Adler 分析阅读）+ 阶段产出持久化
- **阶段 2**: 并行提取（5 个 extractor）
- **阶段 3**: 三重验证筛选（V1 跨域 / V2 预测力 / V3 独特性）+ 优先级评分
- **阶段 4**: 结构化总结撰写（R / I / A1 / A2 / E / B 六段）

## 思想来源

| 来源 | 借鉴内容 |
|---|---|
| Mortimer Adler《如何阅读一本书》 | 阶段 1: 分析阅读四步（结构/解释/批判/适用性） |
| 赵周 RIA 拆书法 | 阶段 4: R-I-A1-A2 基本骨架 |
| Niklas Luhmann Zettelkasten | 原子化 + 用自己的话重写 |
| Tiago Forte Progressive Summarization | 阶段 4 的"可验证压缩链条"思想 |

## 根本洞察

**现有读书方法论都是给人看摘要，不是给人实际使用。**

| 维度 | 给人看摘要 | 给实际使用（content-summary-jw 目标） |
|---|---|---|
| 关键字段 | 故事 / 金句 / 情感钩子 | 可执行步骤 / 边界 / 适用场景 |
| 失败模式 | 读完就忘 | 学了方法论但不知道什么时候用 |
| 成功标准 | 读者"有收获" | 遇到真实问题时能想起并运用 |

所以六段结构（R/I/A1/A2/E/B）中的 A2（应用场景）和 B（边界）是让总结真正有用的关键。

## 重要区分：阶段 1 的四问 vs 阶段 4 的四问

两者形式完全相同（"这本书在谈什么 / 作者说了什么 / 有道理吗 / 与自己的关系"），但**目的和位置不同，不得混淆**:

| | 阶段 1 的四问（Adler 阅读策略） | 阶段 4 的四问（输出导读结构） |
|---|---|---|
| **目的** | 读者向自己提问，以深度理解全书 | 向读者展示总结前的概览导读 |
| **"有道理吗"的具体含义** | 操作者对作者做出批判性评价（知识不足/知识错误/不合逻辑/分析不完整），是阶段 1 的批判段产物 | 概述性评价：有道理的地方 + 需保留批判的地方，供读者快速判断是否值得深入 |
| **产出位置** | 写入 `stage1-understanding.md` 的批判段，**不出现**在 SUMMARY/INDEX 里 | 出现在 SUMMARY.md 和 INDEX.md 的固定前置位置 |
| **谁来回答** | 操作者（执行 skill 的人）基于批判性阅读给出 | 体现在 SUMMARY 的输出内容里，供读者阅读 |

**操作红线**: 阶段 1 的批判性评价（作者的局限/偏见/争议点）不得直接混入阶段 4 的四问输出结构。两者的"有道理吗"维度含义不同。

```
          ┌─────────────────────────────────────┐

          │ + 书籍类型判断                        │  + 阶段产出持久化
          └─────────────┬───────────────────────┘
                        │ stage1-understanding.md
                        ▼
          ┌─────────────────────────────────────┐
          │ 阶段 2: 并行提取                     │  5 个 extractor（可串行回退）
          └─────────────┬───────────────────────┘
                        │ candidates/
                        ▼
          ┌─────────────────────────────────────┐
          │ 阶段 3: 三重验证                     │  V1 跨域 / V2 预测力 / V3 独特性
          │ + 优先级评分                          │  + 质量门控自检

          │ 阶段 4: 结构化总结撰写                │  R / I / A1 / A2 / E / B 六段

                        │
                        ▼

```

详见 `SKILL.md` 的「长书策略」章节。当文本超过 5 万字时，触发分块提取流程。

## 不变量（任何迭代都不能违反）

1. **原子性**: 一个总结单元只做一个方法论，不能"大而全"
2. **可追溯**: 每个单元必须有原文引用，指向源书章节
3. **可验证**: 每个单元必须通过三重验证（V1 + V2 + V3）
4. **反直觉**: 必须体现作者独特的反常识见解，不是复述常识
5. **四问分工**: 阶段 1 批判性评价写入 `stage1-understanding.md`，不混入阶段 4 的 SUMMARY 四问输出结构
