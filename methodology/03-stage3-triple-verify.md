# 阶段 3 — 三重验证筛选 + 优先级评分

## 目标

从候选单元池里，筛出**真正值得独立成篇的方法论单元**，并给出优先级评分。通不过的降级为 example / 引用 / 术语。

这是 content-summary-jw 区别于"书摘工具"的核心质量门。

## 三重验证 (全部通过才录取)

### V1 — 跨域验证 (Cross-domain)

**问题**: 这个单元在书中**至少 2 个独立的语境**下有佐证吗?

- "独立"的含义: 不是同一个案例换个说法,而是两个不同的故事/不同的章节/不同的对象都在讲同一个道理
- **通过**: 《穷查理宝典》中"逆向思维"在投资决策、避免灾难、教学方法三个独立场景中都出现 → 通过
- **不通过**: 某个漂亮句子只在一章里出现过一次,没有书内的独立证据 → 降级为金句 example

**为什么**: 在多个语境中反复出现的,才是作者真正想传达的稳定方法论,而不是一时兴起的表达。

### V2 — 预测力测试 (Predictive Power)

**问题**: 能用这个单元,推导出书里没明说的某个问题的答案吗?

- 自己设计一个书中没直接讨论过的场景
- 尝试用这个方法论去分析它
- **通过**: 能得出一个有意义、非平庸的结论 → 通过
- **不通过**: 只能得出"努力就会成功"之类的废话 → 这个单元没有真正的解释力,降级

**为什么**: 真正的方法论必须有**外推能力**。如果它只能复述书里的例子,它是描述不是方法。

**⚠️ 操作要求**: 在 `novel_question` 字段中，必须显式说明"为什么这个问题书里没有直接讨论"。如果操作者对书的内容不够熟悉而无法判断，应标注"待确认"，不得随意通过。

### V3 — 独特性检验 (Exclusivity)

**问题**: 这个单元是否是"任何聪明人都会说的常识"?

- 如果把作者名字抹掉,一个对书本领域毫无了解的聪明人也能说出来 → 不通过
- 必须是作者**独特视角 / 反直觉见解 / 独特术语体系** → 通过
- **通过**: 段永平的"stop doing list" — 主动列出不做什么,反常识 → 通过
- **不通过**: "要尊重时间" — 太常识了,没人需要一个总结来告诉自己这个

**为什么**: 常识不需要总结承载，Claude 本身就知道。只有作者的**差异化见解**才值得固化。

## 优先级评分

验证通过的方法论单元，按以下维度综合评分（1-5 分）:

| 维度 | 权重 | 评分标准 |
|---|---|---|
| V1 跨域数量 | 25% | 2个独立语境=3分，每增加1个+1分 |
| V2 预测力质量 | 25% | 有意义非平庸结论=4分，有深刻洞见=5分 |
| V3 独特性 | 25% | 有一定独特性=3分，明显反常识=5分 |
| 适用场景广度 | 15% | 仅限单一场景=2分，跨多个领域=4-5分 |
| 执行可行性 | 10% | 有明确可操作步骤=4-5分，理论性强=2-3分 |

**综合评分** = 加权总分（满分 5 分）

在 SUMMARY 篇幅有限时，优先将高分区的方法论单元放在前面，并标注综合评分。

## 验证执行流程

1. 把阶段 2 的 5 个 candidates/*.md 合并成一个总候选池
2. 去重: 同一方法论被多个 extractor 提取的,合并成一条
3. 对每条候选跑 V1 / V2 / V3，记录判断和理由
4. 计算优先级评分（1-5 分）
5. 通过的写入 `books/<slug>/verified.md`，附评分，进入阶段 4
6. 未通过的写入 `books/<slug>/rejected/<id>.md`，**必须写明不通过哪一项及原因**

## 质量门控自检（在进入阶段 4 前执行）

对照 `SKILL.md` 中的四条质量红线，逐项确认:

- [ ] **红线 1**: 每个候选单元都有原文引用（source_quote），且 ≤150 字
- [ ] **红线 2**: 每个候选单元都有完整字段（id/title/type/source_chapter/summary/tags）
- [ ] **红线 3**: V2 的 `why_not_in_book` 字段已填写，不得为空
- [ ] **红线 4**: V3 的 `why_not_common` 字段有实质内容，不是泛泛而言

## 常见失败模式

1. **V1 作弊** — 把同一例子换个说法算两处。要求: 必须是不同章节 + 不同对象 + 不同结论。
2. **V2 作弊** — 用一个其实书里讨论过的类似问题冒充"新问题"。要求: 新问题应该让人第一眼不知道书里怎么说，且 `why_not_in_book` 有实质说明。
3. **V3 过松** — 只要"说得比较文雅"就认为不是常识。要求: 看**内容**本身是否反直觉,而不是措辞。
4. **优先级评分走过场** — 全部给 4 分。应按实际质量打分，高低有别。

## 数量预期

经验上,一本方法论密集的书 (如《穷查理宝典》) 通过率约 30–50%。一本散文类书可能只有 5–10%。通过率过低 (<5%) 或过高 (>80%) 都要警惕:
- 过低: extractor 可能质量差,要重跑
- 过高: 验证标准可能太松
