# 阶段 4 — 结构化总结撰写

## 目标

把阶段 3 通过验证的方法论单元，撰写成结构化的总结文档（SUMMARY.md），并生成全书的 INDEX.md 导航索引。

使用模板: `templates/SUMMARY.md.template`

## RIA 六段

### R — Reading（原文）

- 直接引用 ≤150 字
- 必须标注出处（章节 / 页码 / 段落标识）
- 若原书是英文，引用英文原文 + 你自己翻译的中文，**不要用现成译本**（避免译者版权 + 译本可能失真）

### I — Interpretation（自述）

- 用**你自己的话**重写方法论的核心骨架
- 5–15 行
- 检查: 读完这段，一个没读过原书的人能否理解这个方法论在做什么？若不能，重写。
- 禁止: 照搬原文句子 / 堆砌修辞

### A1 — Past Application（书中案例）

- 作者在书中**亲自**用这个方法论处理过的具体案例
- 至少 1 条，≤3 条
- 每条要点明: 遇到什么问题 → 怎么用这个方法论 → 得出什么结论 → 实际结果如何

这一段的作用是让人在应用这个方法论时，有具体的类比素材可用。

### A2 — Application Scenarios（应用场景）★

**这决定了总结是否真的被人用起来。**

必须明确:
1. **用户在什么情境下会遇到这类问题？**（场景描述，3–5 条）
2. **和相邻方法论有什么不同？**（避免混淆）

### E — Key Points（执行要点）

- 把方法论转成 1-2-3 要点
- 每一步有**可判断的完成标准**
- 如果有判停点（满足 X 条件则跳过某步），显式写出

### B — Boundary（边界）

- 什么时候**不要**使用这个方法论（反场景）
- 作者在书里警告过的失败模式
- 与之相邻但容易混淆的其他方法论

B 的作用是**防止乱用**。没有 B 的方法论总结，会在不该用的时候被用，反而帮倒忙。

## 总结文档结构

每个方法论单元写入 SUMMARY.md，格式:

```markdown
## {{方法论标题}}

### R — 原文
> {{原文引用 ≤150 字}}
> — {{作者}}, {{章节}}

### I — 方法论骨架
{{用自己的话重写, 5-15 行}}

### A1 — 书中案例
#### 案例 1: {{案例名}}
- **问题**: {{作者遇到了什么}}
- **方法论的使用**: {{作者怎么用这个方法论思考}}
- **结论**: {{得出了什么}}
- **结果**: {{实际发生了什么}}

### A2 — 应用场景
{{什么情境下适用}}

### E — 执行要点
1. **{{要点 1}}**
2. **{{要点 2}}**
3. **{{要点 3}}**

### B — 边界
- **不适用**: {{反场景 1}}
- **不适用**: {{反场景 2}}
- **容易混淆**: {{相邻方法论}}
```

## INDEX.md vs SUMMARY.md 的分工

两者功能不同，不得重复:

| | SUMMARY.md | INDEX.md |
|---|---|---|
| **功能** | 逐条展开方法论（完整六段） | 全书导航 + 方法论索引 |
| **四问内容** | 详细展开（完整的"有道理吗"等） | 只保留**四问摘要**（各 1-2 段话），不重复 SUMMARY 的详细展开 |
| **审计信息** | 有（指向 candidates/ 和 verified.md） | 有（指向 candidates/ 和 verified.md） |

**操作要点**: INDEX.md 的"一、这本书在谈什么"到"四、这本书与自己的关系"四问，只写各 1-2 段的摘要；详细展开（如"有道理吗"中分别写"有道理的地方"和"需要批判的地方"）只出现在 SUMMARY.md。

## 锚点一致性要求

SUMMARY.md 目录中的每个方法论锚点，必须与正文中该方法论的 `## {{标题}}` 完全一致，包括标点符号。

**错误示例**:
- 目录写: `[抓住主要矛盾](./SUMMARY.md#抓住主要矛盾)`
- 正文写: `## 主要矛盾与主次转化：抓核心、看变化`

**正确示例**:
- 目录写: `[（五）主要矛盾与主次转化：抓核心、看变化](./SUMMARY.md#五主要矛盾与主次转化抓核心看变化)`
- 正文写: `### （五）主要矛盾与主次转化：抓核心、看变化`

## 质量门控自检（在写入 SUMMARY.md 之前执行）

对照 SKILL.md 中的四条质量红线，逐项确认:

- [ ] **红线 1**: 每个方法论单元都有原文引用，标注了章节位置
- [ ] **红线 2**: 每个方法论单元都有 R/I/A1/A2/E/B 六段，缺一不可
- [ ] **红线 3**: 所有原文引用单段 ≤150 字
- [ ] **红线 4**: 总结体现了作者独特的反直觉见解，不是普通常识
- [ ] **四问板块**: SUMMARY.md 的一至四每问都已填写，无占位符残留
- [ ] **锚点一致**: 目录锚点与正文标题完全一致
- [ ] **INDEX/SUMMARY 分工**: INDEX 不重复 SUMMARY 的详细四问展开内容

## 常见失败模式

1. **I 段写成书摘** — 如果读起来像"本章作者说了 X"，你在抄书不是在解释。重写。
2. **A2 太宽** — "需要决策时"这种描述永远让人不知道该不该用。必须给出**具体可识别的场景**。
3. **E 段只有哲学没有动作** — "保持客观"不是要点，"列出 3 个最不希望发生的结果"才是。
4. **缺 B 段** — 没边界的方法论会被过度使用，最终读者失望。
5. **从 I 直接跳到 E，跳过 A1** — 丢失了"作者亲自用过"的证据，总结失去权威性。
6. **锚点与正文标题不一致** — 读者点击目录链接后无法跳转到正确位置。必须保持完全一致。
