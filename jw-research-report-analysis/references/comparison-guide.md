# 多篇研报对比分析指南

## 对比分析原则

对比的目标不是"总结各方观点"，而是**找到分歧根源、识别共识风险、定位逆向价值**。

### 核心分析框架

```
一致性 → 分歧点 → 分歧根源 → 共识脆弱性 → 逆向价值
```

---

## 2.1 一致性分析方法

问自己：如果把所有报告放在一起，哪些观点是"公理"（无人质疑）？

**提取维度**：
- 行业/公司基本面判断（增长/下行/持平）
- 核心驱动因素认定（哪些因素是各报告都提到的）
- 风险因素共识（哪些风险被反复提及）
- 估值方法论（各报告是否采用相同估值框架）

**输出要求**：
- 每个共识维度标注支持的 report_id
- 不需要所有报告都同意才列——≥2篇即可标注

---

## 2.2 矛盾识别方法

问自己：分析师们在哪些"关键变量"上给出了不同的判断？

**关键变量 ≠ 细节差异**
- "目标价差10%" → 可能是估值参数差异，不一定是矛盾
- "一个看好行业增长、一个认为行业见顶" → 核心矛盾

**分歧根源四分类**：

| 根源类型 | 识别方法 | 典型表现 |
|---------|---------|---------|
| 假设不同 | 对比 hidden_assumptions 中的高脆弱性假设 | 一方假设油价涨，一方假设油价跌 |
| 数据源不同 | 对比 key_arguments 中的 evidence_detail | 引用了不同的行业数据或调研 |
| 分析框架不同 | 对比估值方法或分析逻辑 | DCF vs 相对估值，自上而下 vs 自下而上 |
| 时间窗口不同 | 对比催化剂 timeline | 一方看短期（1Q），一方看中期（1-2年） |

---

## 2.3 假设脆弱性热力图

**这是对比分析中最有价值的输出。**

**构建方法**：
1. 汇总所有报告的 hidden_assumptions
2. 找出≥2篇报告共同依赖的假设
3. 对每个共同假设评估：
   - 共识依赖度：多少篇报告依赖它？
   - 脆弱性：该假设本身有多脆弱？
   - 综合评分：共识依赖度 × 脆弱性

**综合评分矩阵**：

| 共识依赖度 \ 脆弱性 | 高脆弱 | 中脆弱 | 低脆弱 |
|-------------------|--------|--------|--------|
| 高（≥4篇依赖） | **极高风险** | 高风险 | 中风险 |
| 中（2-3篇依赖） | 高风险 | 中风险 | 低风险 |
| 低（1篇依赖） | 中风险 | 低风险 | 忽略 |

**输出格式**：

```json
{
  "assumption": "假设内容",
  "shared_by": ["report_id_1", "report_id_2"],
  "vulnerability_score": "高",
  "sensitivity_analysis": "若该假设不成立，报告1的结论将从'买入'逆转为'中性'，报告2的目标价需下调30%"
}
```

## 2.4 预期差矩阵

**构建方法**：
1. 基于各报告的 expectation_gap 字段
2. 按"挑战共识程度"排序
3. 构建矩阵：

| 报告 | 对共识立场 | 预期差大小 | 争议程度 |
|------|-----------|-----------|---------|
| report_1 | 挑战共识 | 显著 | 高 |
| report_2 | 强化共识 | 微小 | 低 |
| report_3 | 差异化视角 | 中等 | 中 |

**逆向价值判断**：
- 挑战共识 + 显著预期差 = 可能的超额收益来源（如果对了）
- 强化共识 + 预期差微小 = 无超额信息量
- 高争议 = 市场定价不充分的领域

---

## 2.5 操作建议共识

**汇总方法**：
1. 收集各报告的 actionable_advice
2. 计算目标价区间（min, max, 中位数）
3. 汇总仓位建议（一致 / 分歧）
4. 标注操作建议上的关键分歧

**分歧重要性排序**：
- 方向性分歧（买 vs 卖）→ 最高优先
- 仓位分歧（重仓 vs 轻仓）→ 高优先
- 目标价分歧（幅度>20%）→ 中优先
- 期限分歧（短期 vs 长期）→ 低优先
