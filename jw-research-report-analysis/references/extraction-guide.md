# 单篇研报提取指南

## 提取原则

### 证据链规则
每个提取字段必须满足以下条件之一：
1. 直接引用研报原文（带引号标注）
2. 基于原文逻辑链的明确推断（标注"基于XX原文推断"）

不满足以上条件的字段 → 标注"研报未明确表述"或留空。

### 语气判断框架

**五级分类体系**：

| 级别 | 关键词信号 | 典型表述 |
|------|-----------|---------|
| 强烈看多 | "强烈推荐""确定性极强""坚定看好""战略性配置" | 反复使用肯定性极强的措辞 |
| 审慎看多 | "建议关注""维持增持""谨慎乐观""边际改善" | 看多但附带条件或限定词 |
| 中性 | "维持中性""观望""等待信号""分歧仍存" | 不偏不倚或明确表示不确定 |
| 审慎看空 | "短期承压""估值偏高""风险加大""建议规避" | 看空但留有余地 |
| 强烈看空 | "强烈卖出""泡沫""不可持续""立即撤退" | 极度悲观的判断 |

**置信度判断**：

| 置信度 | 信号 |
|--------|------|
| 极度自信 | 大量数据支撑，措辞绝对，无保留意见 |
| 较为确定 | 有数据支撑，措辞肯定，少量风险提示 |
| 模棱两可 | 两种可能性并列，措辞含糊，"可能""或"频现 |
| 相当谨慎 | 大量条件限定，明确表示不确定性高 |

### 隐含假设挖掘法

**挖掘步骤**：
1. 定位报告核心结论（通常在标题、首段、结论段）
2. 追溯支撑核心结论的逻辑链条
3. 对每个逻辑节点问："这个前提如果变了会怎样？"
4. 按脆弱性分级

**脆弱性判断标准**：

| 脆弱性 | 判断条件 | 举例 |
|--------|---------|------|
| 高 | 依赖单一外部变量，该变量波动大或不可控 | "假设原油价格维持在80美元" |
| 中 | 依赖2-3个条件同时成立 | "假设行业增速维持15%且政策不变" |
| 低 | 基于结构性趋势或已确认事实 | "假设城市化率继续提升" |

### 预期差判断法

**步骤**：
1. 推断市场一致预期（基于近期股价表现、行业共识、常见叙事）
2. 提取报告核心立场
3. 对比两者：
   - 观点方向一致 → "强化共识"
   - 观点方向相反 → "挑战共识"
   - 同方向但有独特角度 → "提出差异化视角"

**预期差大小判断**：
- 显著：报告结论与市场主流观点方向相反
- 中等：方向一致但幅度/时间节点显著不同
- 微小：方向一致，仅细节或权重分配不同

### 关键论据提取法（核心论点→分论点→论据）

**三层结构要求**：

1. **核心论点层**：识别支撑报告核心观点的2-4条主要逻辑线。每条核心论点应能独立支撑核心观点的一个维度。
   - 判断标准：如果去掉这条论点，核心观点是否仍然成立？如果不成立，就是核心论点。
   - 核心论点之间应有逻辑区分（不重叠），合在一起应能完整支撑核心观点。

2. **分论点层**：每条核心论点下挂载2-5条分论点，分论点是对核心论点的进一步论证和细化。
   - 分论点必须有独立的论据支撑，不能只有观点没有依据。
   - 分论点之间应有递进或并列关系。

3. **论据层**：每条分论点必须附带——
   - 论据类型：数据 / 逻辑推演 / 政策引用 / 行业调研
   - 论据详情：具体数据、事实或推理过程
   - 原文引用：直接引用研报原文（如有），用引号标注

**反模式**：
- ❌ 核心论点只有1条且过于笼统（如"行业前景好"）
- ❌ 分论点没有论据，只有观点重复（如"看好因为值得看好"）
- ❌ 论据类型全部是"逻辑推演"而无任何数据支撑
- ✅ 核心论点明确，分论点有数据/事实支撑，原文引用到位

### 操作建议提取规则

SKILL.md 1.8 定义了必须提取的6个字段和3条禁止行为，此处补充实操技巧。

**定位技巧**：
- 操作建议通常出现在：研报尾部"投资建议"或"风险提示"前的总结段
- 策略报告的配置建议常以"首推/其次/中期关注"的层次出现
- 个股报告的目标价通常在最后一页的估值表中

**常见陷阱**：
- 分析师用"建议关注"≠"买入"——注意措辞差异
- 目标价有多个估值方法给出不同数字时，取分析师最终采用的那个
- 策略报告给出的行业配置权重 ≠ 个股仓位建议
