# Web Search 板块估值数据获取模式

当 `fetch_market_data.py` 不支持板块级数据（仅支持个股）时，用 web search 获取板块估值分位。

## 搜索策略（漏斗式，3轮）

### 第1轮：广搜定位

```
query: "A股 科技板块 估值 低位 2026年6月"
```
目标：找到哪些板块处于低位、哪些处于高位的初步判断。

### 第2轮：定向确认

```
query: "中证全指软件指数 估值 分位数 2026年6月"
query: "中证动漫游戏指数 估值 分位数 2026年6月"
```
目标：获取具体PE/PB值和分位百分比。

### 第3轮：交叉验证

```
query: "软件板块 估值 百分位 东方财富 同花顺"
```
目标：从多个财经网站交叉验证数据。

## 关键数据字段

每次搜索需提取：
- **指数名称**：如中证全指软件指数、中证动漫游戏指数
- **当前PE/PB**：具体数值
- **估值分位**：如"近一年20%分位"
- **历史区间**：如"历史区间113.77~337.50"
- **相关ETF**：便于用户后续操作

## 输出模板

```
| 板块 | 指数 | 当前PE | 分位数 | 历史区间 | 状态 |
|------|------|--------|--------|----------|------|
| 软件 | 中证全指软件 | 113.77 | 近一年20% | 113.77~337.50 | 低位 |
```

## 已验证的数据源

| 数据源 | 可靠度 | 特点 |
|--------|--------|------|
| 东方财富 | 高 | ETF持仓页有分位数据 |
| 同花顺 | 高 | 概念板块热度排行含估值 |
| 蛋卷基金 | 中 | 指数估值中心，有PE百分位 |
| 券商研报 | 高 | 中信、广发等有系统性估值分析 |

## 注意事项

1. **时效性**：搜索结果中的日期必须检查，优先取最近1周内的数据
2. **口径差异**：不同网站的"分位"可能基于不同时间窗口（近1年/近3年/近5年），需统一口径
3. **盘中vs盘后**：注意区分实时数据和收盘数据
4. **港股vs A股**：同一板块在港股和A股估值可能差异很大，注意区分
