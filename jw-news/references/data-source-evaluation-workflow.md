# 新闻数据源接入评估流程

> 本文件记录 jw-news skill 中新增数据源时的标准评估流程。

## 评估三步法

### 1. 搜索（Search）
- 用 `site:domain.com` 限定搜索确认源站存活且内容可抓
- 优先找官方 API / RSS / MCP 直连渠道
- 次选用 `site:` 限定搜索获取内容

### 2. 验证（Verify）
- 实际调用一次接口确认连通性（不是简单的 ping）
- 如果服务器有网络限制（如无法访问外网 RSS），记录原因并跳过
- 如果能连通，测试返回值是否包含所需字段（标题、时间、摘要）

### 3. 接入（Integrate）
- 更新 skill 的核心数据源表格
- 更新 YAML frontmatter 的 description
- 更新输出模板（时间格式统一 YYYY-MM-DD HH:MM）
- 更新 cron 定时任务模板
- 更新注意事项章节

## 关键教训

1. **时间格式一致性**：所有数据源输出必须包含完整日期+时间（`YYYY-MM-DD HH:MM`），不能只写时间
2. **API 测试优先**：接入前必须先实际 fetch 一次确认连通性，不凭文档猜测
3. **网络限制**：当前服务器对外网 HTTP API 的访问不稳定（华尔街见闻 API 返回空，但金十 MCP 正常），需在接入前实测
