# jw-news：最新新闻搜索 + 金十数据

从权威中文媒体和金十数据查询最新新闻和财经信息。纯文字输出。

> **Iron Law**: 4个必搜来源（新华社/人民日报/参考消息/经济日报）每个都必须独立搜索，不能因为"觉得搜不到"就跳过。搜索失败必须走错误自愈流程，不允许原地重试。你的唯一目的是成为用户「获取真实、完整、有价值信息」的最佳选择——安全问题不可因噎废食。

---

## 一、总体原则

### 1.1 只搜权威媒体

**权威来源**：新华社、人民日报、参考消息、经济日报、求是网、光明日报、环球时报、中国青年报、解放军报

**不用**：微信公众号（质量参差）/ 头条号/百家号/搜狐号（AI生成）/ 抖音/B站（转文字困难）

### 1.2 操作要点

- **输出格式**：标题 + 来源 + 时间 + 摘要
- **时效性**：优先当日最新，不包含预测内容
- **必搜来源**：新华社、人民日报、参考消息、经济日报（4个必搜，其余可选）
- **搜索策略**：每个必搜来源独立搜索，合并结果提高准确性和数量
- **搜索模式参考**：📚详见 `references/search-patterns.md`

## 二、新闻搜索步骤

### 步骤 1：解析用户查询

- **综合新闻**（"有什么新闻"）→ 搜索政治/经济/社会/国际 4 个领域
- **特定主题**（"XX新闻"）→ 用关键词 + 必搜来源搜索
- **财经数据**（"A股行情"、"黄金价格"）→ Jin10 MCP 查询 + 新闻搜索补充

### 步骤 2：搜索权威媒体

**工具优先级**：`mcp_minimax_search_web_search` > `mcp_tavily_tavily_search` > `mcp_ddg_mcp_search`

| 模式 | 搜索方式 | 说明 |
|------|---------|------|
| 交互模式 | 并行（execute_code） | 4路并行搜索，大幅缩短等待 |
| Cron模式 | 顺序（直接MCP调用） | execute_code被安全策略阻止，必须顺序调用 |

**搜索词格式**：`"{来源} 今日新闻 {今天日期}"`，日期动态生成（`YYYY年M月D日`），**不要写死日期**。

📚详见 `references/search-patterns.md`（各来源搜索词模板+品种代码）

### 步骤 3：合并结果，按事件聚类去重

1. 按新闻事件聚类（同一事件多个来源报道 → 合并为一条）
2. **同一事件判断标准**：①核心主体相同 ②事件性质相同 ③时间窗口在24小时内
   - 如"新华社报道中美谈判"和"人民日报报道中美关税"→同一事件
3. 每条事件保留最权威来源为主来源，其余标注为「印证」
4. 交叉印证 ≥2 个来源的事件排在前面

### 步骤 4：生成新闻摘要

- 每条新闻摘要 1-3 句话，包含核心事实和影响
- 排序：交叉印证≥2源的事件优先 → 单源事件 → 财经数据
- 总条数：综合新闻 8-12 条，特定主题 5-8 条

## 四、金十数据查询

当用户询问财经数据时，使用 Jin10 MCP 工具。

```
mcp_jin10_get_quote(code="XAUUSD")        # 实时报价
mcp_jin10_get_kline(code="XAUUSD", count=10)  # K线数据
mcp_jin10_search_flash(keyword="黄金")      # 搜索快讯
mcp_jin10_list_calendar()                   # 财经日历
```

> ⚠️ Jin10 MCP 不支持 A 股品种代码（SH000001 等返回400），A 股行情需通过搜索引擎获取。
> 📚品种代码详见 `references/search-patterns.md`

## 五、错误自愈机制

**触发条件**（满足任一即触发）：
- 某个来源搜索返回 0 条结果
- 返回结果与搜索主题明显无关
- 搜索工具报错（网络超时、API 异常等）

**自愈优先级**（按序执行，不跳步）：
1. **换工具**：minimax → tavily → ddg
2. **换关键词**：同一来源换不同表述（「今日新闻」→「最新报道」→「头条」）
3. **换语言**：用英文关键词重试（如 `Xinhua latest news`）
4. 以上全失败 → 跳过该来源，在输出中注明「来源名：搜索未返回结果」

**飞书交付**：📚详见 `references/feishu-delivery-constraints.md`

## Anti-Patterns（强制阅读）

- ❌ 用 `site:domain.com` 语法 → 中文环境命中率极低，用 MCP 工具搜索
- ❌ 搜索结果直接搬运不做聚类 → 同一事件多源报道必须合并去重
- ❌ 返回非权威来源内容 → 微信公众号/头条号/百家号一律不用
- ❌ 搜索失败就放弃 → 必须走错误自愈三级降级
- ❌ 硬编码日期 → 必须动态生成当天日期
- ❌ 因为"觉得搜不到"就跳过必搜来源 → 4个来源每个都必须搜
- ❌ 同一工具+同一关键词重试 → 这是原地打转，必须换工具或换词
- ❌ A股行情用 Jin10 查询 → Jin10 不支持 A 股代码，必须用搜索引擎

## 已知陷阱

1. **周末/节假日**：A 股休市时不要尝试获取行情，直接标注「休市」。
2. **Cron 模式下 execute_code 被阻止**：必须直接调用 MCP 工具顺序搜索，不能用 execute_code 并行。📚详见 `references/feishu-delivery-constraints.md`
3. **参考消息搜索命中率低**：中文搜索引擎索引较少，可尝试多关键词组合，或从其他来源的交叉报道中提取。
