# 中国宏观经济周期判断系统

> **Iron Law**: 每次分析必须走完 Step 1→2→3→4→5 全流程。数据缺失用最小可用报告模板，不能因为数据不全就跳过周期定位。


基于24本经典著作研究，构建系统化的中国周期判断方法论。

## 快速参考指南

### 核心输出
1. **周期位置判断**：康波、房地产、债务、库存等多周期位置
2. **风险等级评估**：低/中/高/极高
3. **关键信号识别**：3-5个最重要的信号
4. **政策含义分析**：货币政策、财政政策方向
5. **走势特征**：市场含义（仅供参考，不构成投资建议）

### 数据获取优先级
```
web search（推荐） → jin10 MCP 搜索 → tavily web search → 直接爬取统计局官网
```

### 理论框架
详见 `references/theory-compendium.md`（15个理论框架快速索引 + 详细报告链接）

## 执行流程

### Step 1: 数据获取（必做）

**Step 1.0 健康检查**（在获取数据前执行）：

```bash
# 验证 jw-data 脚本可用性（3秒超时）
cd ~/.hermes/skills/jw-investment-data/scripts
timeout 3 python3 fetch_macro.py --schema 2>/dev/null && echo "✅ jw-data OK" || echo "❌ jw-data 不可用，直接走 web search 降级"
```

- 如果健康检查通过 → 走 Step 1.1 脚本获取
- 如果健康检查失败 → 跳过 Step 1.1，直接走 Step 1.2 web search 获取全部指标

**Step 1.1 脚本获取**（健康检查通过时）：

调用 `jw-investment-data` skill 获取宏观全景数据：

```bash
cd ~/.hermes/skills/jw-investment-data/scripts
./jw-data macro all                # 宏观全景（增长+通胀+货币+利率+准备金）
```

**脚本覆盖的指标**（8个端点，自动获取+缓存）：

| 端点 | 指标 | 数据源 | 频率 |
|------|------|--------|------|
| `cpi` | CPI同比/环比 | 东方财富 | 月度 |
| `ppi` | PPI同比 | 东方财富 | 月度 |
| `pmi` | 制造业/非制造业PMI | 东方财富 | 月度 |
| `gdp` | GDP总额/同比/三产 | 东方财富 | 季度 |
| `invest` | 固定资产投资（累计/同比） | 东方财富 | 月度 |

| `fiscal` | 财政收入（累计/同比） | 东方财富 | 月度 |

| `lpr` | LPR 1Y/5Y | 中国货币网 | 月度 |

**需 web search 补充的指标**（脚本未覆盖）：

|------|--------------|--------|---------|

| 美元指数 | jin10 MCP `get_quote("DXY")` 或 web search | — | 实时 |
| 人民币汇率 | jin10 MCP `get_quote("USDCNH")` 或 web search | — | 实时 |
| 国际金价 | jin10 MCP `get_quote("XAUUSD")` | — | 实时 |

**数据发布时间规律**（避免用过期数据）：
```
上旬 7-12日：进出口(海关总署)、外汇储备(外汇局)
中旬 13-18日：社融(央行)、固投/工业/房地产/70城房价(统计局，通常同日发布)
月末：PMI(统计局)、LPR(20日)
季度中：GDP(统计局)
注意：1-2月数据合并发布(3月中旬)
```

**并行搜索策略**（减少 round trip 延迟）：

将 8 个补充指标分为 2-3 批并行搜索，同一批内同时发起：

| 批次 | 指标 | 工具 | 说明 |
|------|------|------|------|
| 批次1 | 社融、进出口、房地产投资/销售、70城房价 | 并行 web search ×4 | 同为统计局/央行月度数据，通常同日发布 |
| 批次2 | 外汇储备、人民币汇率、美元指数、国际金价 | jin10 MCP ×3 + web search ×1 | 实时数据用 MCP，外储用 web search |

- 批次1 和 批次2 可串行执行（先拿月度宏观数据，再拿实时行情）
- 如果某个搜索失败，不阻塞其他指标——该指标标注"数据暂缺"继续
- 每个搜索结果至少保留 2 个来源用于交叉验证

**Jin10 MCP 交叉验证**（提升数据真实性）：
- CPI/PMI 发布前：用 `mcp_jin10_list_calendar()` 获取预期值和前值，与脚本返回的实际值交叉验证
- 社融/外储：用 `mcp_jin10_search_flash("社融")` 或 `mcp_jin10_search_flash("外汇储备")` 获取最新快讯摘要，与 web search 数据交叉验证
- M2：用 `mcp_jin10_search_flash("M2")` 获取最新快讯，与 Baostock 数据交叉验证
- 偏差>5% 时标注⚠️并在报告中说明

**数据验证**：脚本内置交叉验证（M2 双源对比），差异>3%自动标注。web search 补充数据需至少 2 源交叉。

**降级策略**（数据获取失败时）：
1. **脚本超时**（>25秒）：不重试，直接用 web search 获取同指标
2. **数据延迟**（统计局未公布）：使用上期数据 + 标注"数据截至X月"
3. **来源冲突**：优先级为 统计局/央行官网 > 万得/东方财富 > 媒体报道
4. **严重数据缺失**（>50%指标无法获取）：输出最小可用报告（见下方模板）

**最小可用报告模板**（数据严重缺失时）：
```
# 宏观周期判断（数据受限版）
日期：YYYY-MM-DD
⚠️ 数据获取受限，以下为基于已有数据的定性判断。

## 已获取数据
[列出实际获取到的指标和数值]

## 周期位置（定性判断）
- 康波：[基于已有数据的判断]
- 其他周期：[标注"数据不足，无法判断"]

## 风险等级：[基于可用数据推断]

## 补充说明
- 缺失指标：[列出未获取的指标]
- 建议：[何时重新获取/手动补充]
```

**日期有效性检查**（Step 1 末尾必做）：

获取数据后，检查最新数据月份与当前日期的差距：
- 当前日期在上旬（1-10日）→ 最新数据应为上上月（如 6 月应有 4 月数据）
- 当前日期在中旬（11-20日）→ 最新数据应为上月（如 6 月中旬应有 5 月数据）
- 当前日期在下旬（21-31日）→ 最新数据应为上月
- 差距 > 2 个月 → ⚠️ 标注"数据可能滞后"，优先用 web search 获取最新数据
- 差距 > 3 个月 → 🚨 标注"数据严重滞后"，报告中明确说明数据时效性

### Step 2: 周期定位（必做）

依次判断五个周期的位置。每个周期的判断方法和参考指标：

**冲突解决规则**（当不同周期给出矛盾信号时）：
1. **长周期优先**：康波方向 > 房地产方向 > 债务方向 > 库存方向。长周期定基调，短周期定节奏
2. **领先指标优先**：社融拐点 > PMI新订单 > GDP。领先指标的变化比同步/滞后指标更有判断价值
3. **置信度量化**（投票机制）：6 个周期各一票，统计同向票数

| 同向票数 | 置信度 | 输出要求 |
|---------|--------|---------|
| 5-6 票 | 极高(>90%) | 直接输出结论，标注"高度一致" |
| 4 票 | 高(75-90%) | 输出结论，列出1-2个反向信号 |
| 3 票 | 中(60-75%) | 输出结论，明确标注"信号分歧" |
| 2 票 | 低(40-60%) | 不做方向性结论，输出"多空平衡" |
| ≤1 票 | 极低(<40%) | 输出"信号严重分歧，无法判断" |

**长周期权重加成**：康波和房地产同向时，即使短周期反向，仍可给出方向性判断（但置信度降一档）。

**2.1 康波周期**（50-60年）
- 判断方法：检查技术创新阶段 + 商品价格趋势 + 美元周期位置
- 阶段标准：见 `references/周金涛康波理论深度分析报告.md` 的四阶段细分表
- 当前参考：第五次康波(1991至今)，2018年起进入萧条期

**速查阈值**：

| 阶段 | GDP增速 | CPI | 技术创新 | 商品价格 | 美元 |
|------|---------|-----|---------|---------|------|
| 繁荣 | >4% | 1-3% | 扩散期 | 温和上涨 | 稳定 |
| 衰退 | 2-4% | >3% | 成熟期 | 暴涨 | 走弱 |
| 萧条 | <2% | <1% 或通缩 | 空窗期 | 暴跌/底部 | 走强 |
| 回升 | 2-4% | <2% | 萌芽期 | 企稳回升 | 走弱 |

**2.2 房地产周期**（18-20年）
- 判断方法：城市化率拐点 + 人口结构 + 房价/收入比 + 投资/销售面积
- 关键数据：房地产投资增速、70城房价指数、销售面积同比
- 传导机制：见 `references/置身事内_深度研究报告.md` 的土地财政→房价→银行传导链

**速查阈值**：

| 指标 | 上行期 | 顶部信号 | 下行期 | 底部信号 |
|------|--------|---------|--------|---------|
| 房地产投资增速 | >10% | 5-10%且持续放缓 | <0% | 跌幅收窄 |
| 70城房价同比 | >5% | 环比涨幅收窄 | <0% | 跌幅收窄连续3月 |
| 销售面积同比 | >10% | 增速骤降 | <0% | 跌幅收窄 |
| 房价/收入比 | <10 | 10-15 | >15 | 回落至12以下 |
| 城市化率 | <60% | 60-65% | >65% | — |

**2.3 债务周期**
- 判断方法：债务/GDP变化 + 社融效率 + 信贷脉冲（信贷增量占GDP比重的变化率） + 利率空间
- 量化指标：见 `references/达利欧深度研究报告_债务周期与大周期理论_深化补充版.md` 的7阶段量化表
- 实操信号：见 `references/债务周期交易策略_量化数据补充.md` 的货币-信用四象限

**速查阈值**（达利欧7阶段简化版）：

| 阶段 | 债务/GDP | 社融增速 | 信贷脉冲 | 利率空间 | 信号 |
|------|----------|---------|---------|---------|------|
| 早期 | <200% | 与GDP同步 | 正常 | 充足 | 债务≤收入 |
| 泡沫 | 年增20-25% | 远超GDP | 强正 | 收窄 | 债务/GDP>250% |
| 顶部 | >300% | 见顶回落 | 转负 | 极窄 | 信贷问题早于经济峰值 |
| 萧条 | >300% | 骤降 | 强负 | 无空间 | 股价跌50%、GDP降10% |
| 去杠杆 | 高位企稳 | 恢复 | 弱正 | 有限 | 年印钞GDP的4% |

**货币-信用四象限**（明明框架）：

| 象限 | 货币(M2) | 信用(社融) | 资产含义 |
|------|----------|----------|---------|
| 宽货币+紧信用 | 宽松 | 收缩 | 利好债券 |
| 宽货币+宽信用 | 宽松 | 扩张 | 利好股票 |
| 紧货币+宽信用 | 收紧 | 扩张 | 利好商品 |
| 紧货币+紧信用 | 收紧 | 收缩 | 利好现金 |

**2.4 库存周期**（基钦周期，3-4年）
- 判断方法：PMI + 产成品库存 + 工业增加值
- 四阶段：主动补库→被动补库→主动去库→被动去库
- 领先信号：PMI新订单-产成品库存差

**速查阈值**：

| 阶段 | PMI新订单-产成品库存 | PMI | 工业增加值 | 库存 | 特征 |
|------|-------------------|-----|----------|------|------|
| 主动补库 | >0 | >50且↑ | ↑ | ↑ | 需求强+企业主动加库存 |
| 被动补库 | <0 | >50但↓ | ↓ | ↑ | 需求弱+库存积压 |
| 主动去库 | <0 | <50 | ↓ | ↓ | 需求弱+企业主动减库存 |
| 被动去库 | >0 | <50但↑ | ↑ | ↓ | 需求恢复+库存不足 |

**2.5 市场情绪**
- 定量：PE、信用利差、融资融券余额、北向资金
- 定性：马克斯量体温法（11组对比指标）— 见 `references/马克斯周期理论深度分析报告.md`
- 行为偏差检查：卡尼曼14种认知偏差（过度自信、锚定、可得性偏差等）— 见 `references/卡尼曼思考快与慢_深度研究报告.md`
- 逆向检验：芒格"反过来想"——当前共识可能是错的证据是什么？— 见 `references/穷查理宝典_深度研究报告.md`

**2.6 朱格拉周期（产能周期，9-10年）**
- 判断方法：产能利用率 + 制造业投资增速 + 工业企业利润
- 关键数据：产能利用率（统计局季度）、制造业固定资产投资增速、工业企业利润总额同比
- 传导机制：产能过剩→价格战→利润压缩→去产能→供需再平衡→新一轮扩张

**速查阈值**：

| 指标 | 扩张期 | 顶部信号 | 收缩期 | 底部信号 |
|------|--------|---------|--------|---------|
| 产能利用率 | >78% | >82%且持续 | <75% | <72%且企稳 |
| 制造业投资增速 | >10% | 5-10%且放缓 | <5% | 跌幅收窄转正 |
| 工业企业利润同比 | >10% | 增速骤降 | <0% | 跌幅收窄 |
| PPI | 温和上涨 | 加速上涨 | 下跌 | 跌幅收窄 |
| PMI新订单 | >52 | 50-52徘徊 | <49 | 回升至50以上 |

**朱格拉 vs 库存周期区分**：朱格拉看产能（供给侧，9-10年），库存看需求侧（3-4年）。当两者同处于收缩期时，经济下行压力最大。

### Step 3: 风险评估

**3.1 灰犀牛识别**
- 5阶段判断标准：见 `references/灰犀牛_深度量化补充报告.md`
- 中国灰犀牛清单（每次用最新数据判断阶段，不读快照）：

| 灰犀牛 | 判断阶段的指标 | 数据源 |
|--------|--------------|--------|
| 房地产持续下行 | 房地产投资增速连续月数为负、70城房价同比连续下跌月数 | 统计局 |
| 地方政府债务风险 | 城投债利差、地方政府隐性债务规模变化、专项债发行节奏 | web search |
| 内需不足/消费低迷 | 社零增速、消费者信心指数、居民存款增速(反向) | 统计局 |
| 人口老龄化加速 | 出生率、老龄化率、劳动年龄人口变化 | 统计局 |
| 中美关系紧张 | 关税政策变化、科技制裁清单、外交事件频率 | web search |

- 阶段判断标准（量化触发条件）：

| 灰犀牛 | ①否认 | ②得过且过 | ③争论 | ④惊恐 | ⑤行动/崩溃 |
|--------|--------|----------|--------|--------|-----------|
| 房地产 | 投资增速连续3月为负但官方定调"调整" | 连续6月为负，出台零星政策 | 连续12月为负，多部委联合讨论 | 投资增速<-10%，房企大面积违约 | 系统性救市（PSL/收储/限购全面取消） |
| 地方债务 | 城投债利差走扩但不提风险 | 利差>200bp，个别省展期 | 利差>300bp，中央发文化债 | 城投债违约/技术性违约 | 特殊再融资债大规模发行 |
| 内需不足 | 社零增速<5%但定调"结构优化" | 连续6月<4%，消费券零星发放 | 连续12月<3%，多部委促消费 | 社零<2%，消费者信心<85 | 大规模消费刺激（如家电下乡2.0） |
| 人口老龄化 | 出生率<10‰但不提危机 | 出生率<8‰，延迟退休讨论 | 出生率<7‰，养老金改革方案出台 | 劳动年龄人口年减>500万 | 全面放开生育+移民政策 |
| 中美关系 | 关税加征但规模有限 | 科技制裁清单扩大，外交降级 | 贸易额同比下降>15%，脱钩加速 | 金融制裁/SWIFT威胁 | 全面经济对抗 |

**3.2 黑天鹅评估**（低概率高冲击事件）
- 系统脆弱性维度：杠杆水平、冗余充足度、同质化程度、去波动化程度
- 肥尾风险：见 `references/肥尾效应_深度量化补充报告.md` 的κ统计量方法

**3.3 反脆弱策略**（从波动中获益的系统设计）
- 杠铃策略：85-90%极安全 + 10-15%极冒险
- 详见 `references/反脆弱_深度量化补充报告.md`

### Step 4: 中国特色分析

**4.1 城乡二元结构**（温铁军）
- 农村承载能力：城镇化率（<65%仍有缓冲）、农民工返乡规模（春运数据）、农村居民人均可支配收入增速
- 危机转嫁空间：城乡收入比（>2.5=有空间）、农村人口占比（>30%=有缓冲）
- 详见 `references/温铁军八次危机-深度分析报告.md`

**4.2 政府-经济关系**（兰小欢）
- 土地财政：土地出让金/地方财政收入（>50%=高度依赖）、房地产相关税收占比
- 城投平台：城投债存量规模、城投债信用利差、地方政府隐性债务估算
- 产业政策：专项产业基金规模、产能过剩行业投资增速
- 详见 `references/置身事内_深度研究报告.md`

**4.3 中央-地方关系**
- 财政赤字结构：中央财政赤字率 vs 地方专项债规模
- 地方投资热度：各省固投增速标准差（>5%=分化严重）
- 隐性债务：城投有息负债增速、非标违约事件数

**4.4 三大赤字同步性**
- 财政赤字：一般公共预算赤字率
- 金融赤字：商业银行不良率、影子银行规模变化
- 外汇赤字：资本和金融账户差额、外储变动
- ⚠️ 三者同方向恶化 = 系统性风险信号

**4.5 信任变量**（奥尔德雷德/卡尼曼）
- 政策可信度：政策承诺vs实际执行偏差率、"政策翻烧饼"频率
- 市场信心：消费者信心指数、企业家信心指数、PMI生产经营预期
- 详见 `references/通往衰败之路_深度量化补充报告.md`

### Step 5: 综合输出（必做）

按 `templates/report-template.md` 输出完整报告，包含：
1. 核心结论（先行）
2. 长周期定位（康波+房地产）
3. 中周期诊断（朱格拉+债务）
4. 短周期跟踪（库存+市场情绪）
5. 外部环境（美元+资本流动）
6. 中国特色分析
7. 风险预警（灰犀牛+黑天鹅）
8. 政策含义（货币+财政）
9. 走势特征与市场含义（仅供参考，不构成投资建议）
10. 关键数据表

## 注意事项

### 核心原则
1. 周期不可消除，拐点不可精确预测
2. 多周期共振时产生深度萧条
3. 中国特殊性：城乡二元结构、政策主导性强
4. 信任变量是周期的隐形变量
5. 所有分析仅供参考，不构成投资建议

### 分析技巧
1. 先看趋势，再看绝对值
2. 关注领先指标（PMI新订单、社融、房地产销售）
3. 注意季节性因素（春节效应、季末效应）
4. 结合政策信号判断方向
5. 用卡尼曼偏差清单检查自己的判断是否受认知偏误影响
6. 用芒格逆向思维——如果当前共识是错的，证据是什么？

### 常见问题

**Q: 数据不一致怎么办？**
A: 优先官方数据，多源交叉验证，关注数据修正（初步值→修正值→终值）。

**Q: 如何判断周期拐点？**
A: 关注领先指标拐点 + 政策转向 + 马克斯"投降机制"（最后一个抵抗者投降即为拐点）。

## Anti-Patterns（强制阅读）

- ❌ 用当期单月数据判断长周期（康波/房地产）→ 必须看趋势（≥6个月数据）
- ❌ 忽视季节性因素（春节效应/季末效应）→ 1-2月数据合并看
- ❌ 灰犀牛阶段用定性描述代替量化触发 → 必须对照5阶段量化表
- ❌ 周期信号矛盾时不做置信度量化 → 必须走投票机制（6周期各一票）
- ❌ 数据缺失就放弃 → 走最小可用报告模板
- ❌ 硬编码"当前周期位置" → 每次用最新数据重新判断
- ❌ 长短周期矛盾时只看短周期 → 长周期定基调，短周期定节奏

## Pitfalls

### Pitfall 0: references 目录健康检查（去重+孤立文件+判断日志）
多轮研究会产出同主题多版本文件。每轮完成后执行：
1. `ls references/ | sort` 检查重复版本，保留最新
2. **孤立文件检查**：遍历 references/ 下所有 .md 文件，用 grep 验证每个文件至少在 SKILL.md 或 theory-compendium.md 中被引用一次。2026-06-04 实测 45% 文件未被引用（已修复至 0%）。孤立文件要么添加到参考资料索引表，要么删除
3. 每次宏观判断后记录到 `references/judgment-log.md`，每季度回测校准

### Pitfall 1: 批量研究的 429 限流
delegate_task 并行读大文件易触发 429。**每批最多 2 个并行子任务**（2026-06-04 实测 12 本书 6 批零失败）。

### Pitfall 2: skill_manage patch 名称精确匹配
`skill_manage(action='patch', name='macro-analysis')` 会报错，必须用 `jw-macro-analysis`。

### Pitfall 3: patch 重复条目
patch 前用 `grep -c "old_string" SKILL.md` 验证唯一性。old_string 不够唯一会导致重复条目。

### Pitfall 4: 书籍研究增量原则
子任务 prompt 指定已有 reference 路径，要求"深化补充而非重复"。深度比 ≤40% 需补充，≥70% 可跳过。

### Pitfall 5: 批量研究四阶段模式
阶段一(覆盖分析) → 阶段二(并行执行，每批≤2个) → 阶段三(整合去重) → 阶段四(SKILL.md更新)

### Pitfall 6: SKILL.md 膨胀后的结构优化模式
**问题**：SKILL.md 随多轮研究不断膨胀（本轮从 1120 行开始），理论内容占比超过执行指引，导致 agent 加载时浪费 token 且找不到执行入口。

**优化三板斧**（2026-06-04 两轮优化验证）：
1. **理论下沉**：理论框架章节迁移到 `references/theory-compendium.md`，SKILL.md 只保留执行指引 + 指标速查表
2. **去重合并**：references 目录同主题多版本文件（如"深度分析"+"量化补充"+"优化补充"）按"保留最新 1-2 个"原则清理。**清理前必须用 grep 检查 SKILL.md 和 theory-compendium.md 的引用**，否则会产生断裂引用
3. **静态→动态**：硬编码的"当前状态"快照（如"当前中国周期位置"）改为判断框架 + 参考指标表，每次执行时用最新数据重新判断

**降级策略模板**（Step 1 数据获取失败时）：
- 搜索无结果 → 换关键词重试
- 数据延迟 → 用上期数据 + 标注时效性
- 来源冲突 → 官方源优先 + 标注差异

**冲突解决模板**（Step 2 周期信号矛盾时）：
- 长周期优先（康波 > 房地产 > 债务 > 库存）
- 领先指标优先（社融 > PMI > GDP）
- 多源交叉（3+周期同向=高置信，各说各话=标注"信号分歧"）

### Pitfall 7: 不要在 skill 内部复制另一个 skill 的逻辑
**问题**：SKILL.md 中自建数据获取脚本（如 akshare 调用、web search 关键词列表），与 `jw-investment-data` 等已有 skill 功能重复。

**解决方法**：数据获取通过 `jw-investment-data` skill（`./jw-data macro all`），SKILL.md 只写调用指令。补充数据用 web search + jin10 MCP。不要在本 skill 内创建 scripts/。

### Pitfall 8: 不要给分析模式分级

不要设置 Quick/Standard/Focus 等模式选择。用户每次询问都期望全量分析，模式选择是多余复杂度。去掉 Step 0 模式选择，每次走完 Step 1→2→3→4→5 全流程。

### Pitfall 9: 删除内容后必须全文扫描残留引用

删除任何章节/模式/术语后，用 `grep -n "被删关键词" SKILL.md templates/*.md` 扫描全文，确保没有其他位置还在引用已删除的内容。

**案例**（2026-06-04 审计发现）：Pitfall 8 删除了 Quick/Standard/Focus 模式，但 Step 5 仍引用"Standard 模式模板"——与 Pitfall 8 自身矛盾。修复：删除模式引用，改为直接引用模板文件名。

### Pitfall 10: SKILL.md 与模板措辞必须三方同步

SKILL.md 的输出列表、模板的选填字段头、模板的章节标题——三处措辞必须一致。任何一处修改后，用 `grep` 检查另外两处是否需要同步。

**案例**（2026-06-04 审计发现）：模板第九章标题改为"走势特征与市场含义"，但 SKILL.md Step 5 列表仍写"走势特征"，模板选填字段头仍写"走势特征（仅供参考，不输出配置比例）"。三处措辞不一致。

### Pitfall 11: 重构后检查领域术语定义覆盖

大规模重构（>20%行变更）后，用 `grep` 检查所有在决策树/阈值表中使用的领域术语，确认每个术语在首次使用前有定义或括号注释。无定义的术语会让 agent 歧义执行。

**检查方法**：`grep -n "术语" SKILL.md` — 如果首次出现没有"（定义）"或"："解释，就需要补充。
**案例**（2026-06-04 审计发现）：基钦、黑天鹅、反脆弱、信贷脉冲 4 个术语在 Step 2/3 使用但无定义。
