# 行业分析框架指南（Industry Analysis Framework Guide）

本文件提供行业分析的标准框架、分析方法和信贷场景特殊要求，供 jw-industry-analysis Skill 在步骤1-8执行时查阅。

---

## 一、行业分析标准框架

### 1.1 分析框架选择矩阵

| 行业类型 | 核心框架 | 辅助框架 | 关键指标 |
|---------|---------|---------|---------|
| 传统制造业 | 产业链分析 + 成本曲线 + 产能周期 | PEST、波特五力 | 产能利用率、CR5、毛利率 |
| 科技行业 | 技术成熟度曲线 + 渗透率S曲线 + 创新扩散模型 | PEST、价值链分析 | 研发投入占比、专利数量、渗透率 |
| 消费行业 | 消费者画像 + 渠道分析 + 品牌力评估 | PEST、竞争格局分析 | 客单价、复购率、品牌集中度 |
| 周期性行业 | 供需分析 + 库存周期 + 价格弹性 | PEST、波特五力 | 库存/销售比、价格弹性系数、β值 |
| 新兴行业 | 商业模式画布 + 政策环境分析 + 技术路线对比 | PEST、情景分析 | 政策补贴力度、技术成熟度、现金流模型 |

### 1.2 行业生命周期判定标准

| 生命周期阶段 | 渗透率 | 增速 | 竞争格局 | 盈利状况 | 典型行业 |
|------------|--------|------|---------|---------|---------|
| 导入期 | <5% | 不稳定 | 技术路线未定 | 普遍亏损 | 氢能、量子计算 |
| 成长期 | 5%-30% | >20% | 龙头开始显现 | 逐步改善 | 光伏、新能源车 |
| 成熟期 | >50% | <10% | 集中度提升 | 稳定盈利 | 钢铁、水泥 |
| 衰退期 | 下降 | 负增长 | 产能出清 | 普遍亏损 | 传统燃油车、煤炭 |

### 1.3 市场规模测算方法

**方法1：自上而下（TAM/SAM/SOM）**

```
TAM（总可及市场）= 全球/全国市场总规模
SAM（可服务市场）= TAM × 目标市场占比（地理/产品/客户细分）
SOM（可获得市场）= SAM × 预期市场份额
```

**方法2：自下而上**

```
市场空间 = 终端需求量 × 单价
```

**方法3：供给侧验证**

```
市场空间 = 总产能 × 产能利用率 × 单价
```

**交叉验证要求**：
- 至少使用2种方法测算
- 结果差异 > 20% 须说明原因
- 明确标注假设条件和数据来源

## 二、产业链分析方法

### 2.1 产业链拆解模板

```
上游（原材料/核心零部件）
  ↓ [成本传导机制]
中游（制造/加工/集成）
  ↓ [渠道/定价模式]
下游（渠道/终端应用/消费者）
```

**各环节分析要点**：

| 环节 | 分析维度 | 关键问题 |
|------|---------|---------|
| 上游 | 资源禀赋、技术壁垒、集中度 | 原材料价格波动？供给弹性？ |
| 中游 | 制造工艺、规模效应、成本控制 | 产能利用率？毛利率水平？ |
| 下游 | 渠道分布、品牌力、客户集中度 | 议价能力？需求弹性？ |

### 2.2 价值链利润池分析

**利润迁移判断标准**：

| 利润迁移方向 | 判断依据 | 典型场景 |
|------------|---------|---------|
| 向上游集中 | 上游毛利率 > 中游 + 10pct，且持续扩大 | 锂矿 > 电池 > 整车 |
| 向下游集中 | 下游毛利率 > 中游 + 10pct，且持续扩大 | 品牌 > 代工 |
| 向中游集中 | 中游规模效应显著，成本优势突出 | 台积电代工模式 |

### 2.3 上下游博弈关系

**议价权判断矩阵**：

| 维度 | 上游强势 | 下游强势 |
|------|---------|---------|
| 集中度 | CR3 > 60% | CR3 > 60% |
| 转换成本 | 高（技术壁垒/认证周期长） | 高（品牌依赖/渠道控制） |
| 差异化程度 | 高（不可替代） | 高（渠道稀缺） |
| 成本传导 | 能传导到中游 | 能传导到中游 |

## 三、竞争格局分析方法

### 3.1 波特五力定量评估

**评分标准（1-5分）**：

| 竞争力量 | 1分（极弱） | 3分（中等） | 5分（极强） |
|---------|-----------|-----------|-----------|
| 现有竞争者 | CR5 > 80%，价格稳定 | CR5 = 40%-60%，价格战偶发 | CR5 < 30%，价格战持续 |
| 潜在进入者 | 壁垒极高（牌照/技术/资本） | 壁垒中等（需要一定投入） | 壁垒极低（谁都能进） |
| 替代品威胁 | 无替代品 | 有替代但成本高 | 替代品成熟且便宜 |
| 供应商议价 | 供应商分散，转换容易 | 供应商集中，但可替代 | 供应商垄断，无可替代 |
| 买方议价 | 买方分散，产品差异化 | 买方集中，但产品必需 | 买方垄断，可随意压价 |

### 3.2 市场集中度指标

**CRn（行业前n家企业市场份额之和）**：

```
CR5 = 前5大企业营收 / 行业总营收 × 100%
```

**判定标准**：

| CR5 | 竞争格局 | 行业特征 |
|-----|---------|---------|
| > 80% | 极高寡占型 | 电信、烟草 |
| 60%-80% | 高寡占型 | 汽车、钢铁 |
| 40%-60% | 低寡占型 | 家电、水泥 |
| 20%-40% | 竞争型 | 餐饮、服装 |
| < 20% | 原子型 | 农业、零售 |

**HHI指数（赫芬达尔-赫希曼指数）**：

```
HHI = Σ(各企业市场份额²)
```

**判定标准**：

| HHI | 竞争格局 | 监管态度 |
|-----|---------|---------|
| > 2500 | 高度集中 | 严格审查并购 |
| 1500-2500 | 中度集中 | 关注并购影响 |
| < 1500 | 竞争充分 | 一般不干预 |

## 四、信贷场景专属要求

### 4.1 行业授信准入评估要点

| 评估维度 | 关注重点 | 红线信号 |
|---------|---------|---------|
| 行业准入合规性 | 是否属于限制类/淘汰类产业 | 列入发改委《产业结构调整指导目录》限制类或淘汰类 |
| 产能周期位置 | 当前处于扩张期还是过剩出清期 | 行业产能利用率 < 70%；在建产能 / 现有产能 > 30% |
| 政策敏感度 | 行业是否依赖政策补贴 | 补贴占利润总额 > 30%；政策窗口期 ≤ 1年 |
| 竞争格局稳定性 | 行业集中度变化方向 | CR5 < 20% 且持续下降；价格战持续超2年 |
| 周期波动幅度 | 行业景气波动对企业收入/利润的传导强度 | 历史周期谷底营收降幅 > 30%；价格弹性系数 > 2 |
| 下游集中度风险 | 主要终端客户集中度是否过高 | 前三大客户营收占比 > 50% 为行业普遍现象 |

### 4.2 行业评级标准

| 评级 | 定义 | 授信政策 |
|------|------|---------|
| 推荐准入 | 行业景气向上，竞争格局稳定，无红线触发 | 正常授信，优先支持 |
| 审慎准入 | 行业景气平稳，存在部分风险但可控 | 授信需加强审批，控制额度 |
| 限制准入 | 行业景气向下，触发1-2条红线 | 暂停新增授信，存量逐步压降 |
| 禁止准入 | 行业严重过剩或淘汰类，触发3条以上红线 | 全面退出，不得新增 |

### 4.3 典型应用场景分析重点

| 场景 | 分析重点 | 特殊关注 |
|------|---------|--------|
| 行业授信准入评估 | 行业红线扫描 + 产能周期 + 政策方向 | 对标发改委产业目录；产能利用率 |
| 授信政策年度调整 | 景气变化 + 竞争格局演变 + 风险矩阵更新 | 与上年对比的边际变化 |
| 特定客户行业背景分析 | 客户所在细分赛道竞争位置 + 盈利可持续性 | 该行业龙头与尾部的利润差异 |
| 不良贷款成因分析 | 行业系统性风险 vs 个体经营风险区分 | 同类企业整体财务趋势对比 |
| 新兴行业准入研究 | 技术成熟度 + 政策支持力度 + 商业模式可行性 | 渗透率 S 曲线位置；现金流模型 |

## 五、数据源与质量要求

### 5.1 数据源分级

| 级别 | 数据源 | 适用场景 | 可信度 |
|------|--------|---------|--------|
| 一级 | 国家统计局、央行、上市公司公告 | 官方数据引用 | 高 |
| 二级 | 行业协会、工信部、发改委 | 行业统计数据 | 中高 |
| 三级 | Wind、同花顺、商业研报 | 市场数据、预测数据 | 中（须标注机构） |

### 5.2 数据质量要求

1. **行业边界可追溯**：分析对象须对应 GB/T 4754-2017 行业代码，报告首页列明
2. **市场规模测算逻辑透明**：TAM/SAM/SOM 或供需双侧测算须列出公式和假设
3. **竞争格局量化**：CR3/CR5 及 HHI 为必选指标，无数据时须说明原因并给出估算区间
4. **风险矩阵完整度**：风险矩阵须覆盖政策/技术/供需/竞争4个维度，每维度至少1条风险事项
5. **情景分析差异性**：乐观/中性/悲观三情景的核心假设须有实质差异（增速区间至少相差5pct）
6. **一票否决自检**：输出前须对6条行业红线逐项检查，触发须标注并说明影响程度
7. **结论层级清晰**：报告须明确给出行业评级（推荐准入/审慎准入/限制准入/禁止准入），有量化支撑

## 六、常见行业分析陷阱

### 6.1 行业边界过泛

**症状**：分析"制造业"或"批发零售业"，缺乏针对性

**解决**：始终查询客户精确行业代码，对照 GB/T 4754-2017 细化至中类或小类

### 6.2 市场规模测算逻辑不透明

**症状**：仅引用第三方研报数字，未说明测算口径和假设

**解决**：必须列出公式和假设，禁止仅引用第三方数字而不说明口径

### 6.3 竞争格局未量化

**症状**：仅描述"竞争激烈"，未给出 CR5、HHI 等量化指标

**解决**：CR3/CR5 及 HHI 为必选指标，无数据时须说明原因并给出估算区间

### 6.4 情景分析无差异

**症状**：乐观/中性/悲观三情景的增速假设仅相差1-2pct

**解决**：三情景的增速区间至少相差5pct，核心假设须有实质差异

### 6.5 红线信号被淹没

**症状**：行业产能利用率 < 60%，但红线提示未放在报告开头

**解决**：红线核查完成后，如有任何触发，必须在报告开头单独成节标注
