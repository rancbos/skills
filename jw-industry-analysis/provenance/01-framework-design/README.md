# 步骤1：分析框架设计

## 输入
- PEST 宏观环境分析框架
- 波特五力竞争模型
- 产业链价值链拆解方法论
- TAM/SAM/SOM 市场规模测算框架
- 情景分析法（乐观/中性/悲观）
- 贷前行业准入评估标准

## 过程
1. 研究经典行业分析框架在银行信贷场景的适用性
2. 设计分析框架选择指南（传统制造业/科技行业/消费行业/周期性行业）
3. 定义行业生命周期判定标准（导入期/成长期/成熟期/衰退期）
4. 建立竞争格局量化指标体系（CR3/CR5/CR10、HHI指数）
5. 设计供需平衡表推演方法

## 输出
- 行业分析框架选择指南（已移至 references/industry-analysis-guide.md）
- 行业生命周期判定标准（已整合至 SKILL.md 步骤2）
- 竞争格局量化指标体系（已整合至 SKILL.md 步骤4）
- 供需平衡表推演模板（已移至 assets/industry-analysis-template.md）

AI架构师团队 + 行业研究员

## 关键发现
1. 不同行业类型需要不同的分析框架组合，不能一刀切
2. 信贷场景更关注行业准入合规性和产能周期位置，而非投资回报率
3. 竞争格局必须量化（CR5、HHI），禁止纯定性描述
4. 情景分析必须有实质差异（增速区间至少相差5pct）
