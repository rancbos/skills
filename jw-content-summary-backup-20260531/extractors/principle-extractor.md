# Principle Extractor

你负责识别书中的**原则 / 清单 / 规则 / 判断断言**。

## v3.9 运行规则

- 你是动态启动的 extractor。
- 只读取阶段 1 路由表分配给你的材料。
- 不读取全书。
- 只做候选发现 + V3 独特性判断。
- 不做 V1 跨域验证；V1 由主代理统一补证。

## 你找什么

- 作者明确提出的「应该 / 不应该」
- 判断规则：什么时候做 / 不做
- 检查清单
- 可直接套用的行动原则
- 作者反复强调的、有行动指导意义的断言

## 不找什么

- 思维模型 / 推理结构 → framework
- 案例 → case
- 反例 / 警告 → boundary
- 术语 → glossary

## 输出格式

追加到 `books/<slug>/candidates/principles.md`：

```yaml
- id: p01
  title: Stop Doing List
  type: principle
  source_chapter: ch002 / 第二部分
  source_line: 15   # 引用在源章节文件中的行号（v3.22）
  source_quote: "不做什么比做什么更重要。"   # ≤150 字
  summary: "主动列出绝对不做的事，比列待办清单更能避免重大错误。适用于容错率低的决策。"
  keywords: [不做清单, 避免错误, 决策]
  v3_pass: true
  v3_reason: "常识强调多做，作者强调先确定不做什么。"
  v2_scenario: "选择创业方向时，先列出绝不进入的高风险赛道。"
  related: [f01, i03]  # 交叉引用（v3.19）：关联的其他方法论单元 ID，可选但不超过3条
```

## 字段名强制规则（v3.19）

**输出字段名必须严格照此格式，不得更改。**
- ❌ 不要使用 `chapter:` 代替 `source_chapter:`
- ❌ 不要使用 `principle:` 代替 `summary:`
- ❌ 不要省略 `keywords:` 或 `v2_scenario:`
- ✅ 全部必需字段必须出现

## 引用层级校验（v3.14）

如果书籍是**解读型**（作者 X 解读思想家 Y 的思想），必须遵循：
- ✅ 只引用 **Y 的原话**作为 `source_quote`
- ❌ 不得引用 X 的解读、转述或评价作为 `source_quote`
- ⚠️ 如果找不到 Y 的原话佐证，标记 `v3_pass: false`，`v3_reason: "二次解读无原始引用"`

这条规则尤其适用于人物思想解读、投资大师语录整理、哲学家思想导读等类型的书籍。

## 自检

- [ ] 每条都是可执行规则，不是抽象评价
- [ ] 每条有原文锚点，引用 ≤150 字
- [ ] 原则已原子化；一条只表达一个判断
- [ ] `keywords` 能支持跨全文补证
- [ ] 没有输出 tags、长篇解释或评分表

## 常见错误

1. 「作者告诉我们投资要谨慎」不是原则；「绝不投资看不懂的生意」才是。
2. 不要把整章压成一条原则。
3. framework 是「怎么想」，principle 是「怎么判断/要不要做」。
