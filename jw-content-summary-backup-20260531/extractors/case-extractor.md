# Case Extractor

你负责识别作者在书中使用的**具体案例 / 故事 / 实践场景**。

## v3.9 运行规则

- 优先读取 `snippets/cases.json`。
- 只在案例上下文不足时读取路由分配的章节片段。
- 不读取全书。
- 只做候选发现 + V3 独特性判断。
- 不做 V1 跨域验证。

## 你找什么

- 作者亲自讲述的案例
- 用来证明某个方法论有效的故事
- 实验、历史事件、商业案例、个人经历
- 案例中清楚包含：问题 → 行动/方法 → 结果

## 不找什么

- 纯轶事但无方法论含义
- 只有观点没有情境的段落
- 失败/反面案例 → boundary

## 输出格式

追加到 `books/<slug>/candidates/cases.md`：

```yaml
- id: c01
  title: {{案例标题}}
  type: case
  source_chapter: ch005 / 第五章
  source_line: 10   # 引用在源章节文件中的行号（v3.22）
  source_quote: "{{≤150 字原文}}"
  summary: "问题是什么；作者/人物采取了什么做法；结果是什么；它支撑哪个方法论。"
  keywords: [案例关键词, 方法论关键词]
  linked_method_hint: "可能支撑：逆向思维"
  v3_pass: true
  v3_reason: "案例展示了作者区别于常规做法的行动逻辑。"
  v2_scenario: "相似逻辑可迁移到：{{新场景}}"
  related: [f01, p01]  # 交叉引用（v3.19）：关联的方法论单元 ID，不超过3条
```

## 字段名强制规则（v3.19）

**输出字段名必须严格照此格式，不得更改。**
- ❌ 不要使用 `chapter:` 代替 `source_chapter:`
- ❌ 不要使用 `case:` 代替 `summary:`
- ❌ 不要省略 `keywords:` 或 `v2_scenario:`
- ✅ 全部必需字段必须出现

## 引用层级校验（v3.14）

如果书籍是**解读型**（作者 X 解读思想家 Y 的思想），必须遵循：
- ✅ 只引用 **Y 的原话**作为 `source_quote`
- ❌ 不得引用 X 的解读、转述或评价作为 `source_quote`
- ⚠️ 如果找不到 Y 的原话佐证，标记 `v3_pass: false`，`v3_reason: "二次解读无原始引用"`

这条规则尤其适用于人物思想解读、投资大师语录整理、哲学家思想导读等类型的书籍。

## 自检

- [ ] 有明确人物/场景/问题/行动/结果
- [ ] 不是空泛故事
- [ ] 指明它可能支撑哪个方法论
- [ ] 引用 ≤150 字
- [ ] 没有长篇复述案例全文
