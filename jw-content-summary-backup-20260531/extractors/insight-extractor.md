# Insight Extractor

你负责识别散文/随笔/思想类书籍中的**独特洞见**。

## 何时启动

阶段 1 判定为：散文/随笔集、思想札记、非系统化论述，但存在高密度洞见时启动。

## 你找什么

- 反直觉判断
- 区别于常识的观察
- 可迁移到其他领域的洞见
- 作者对某个问题的独特视角

## 不找什么

- 普通金句
- 情绪表达
- 没有论证支撑的漂亮句子
- 完整框架 → framework
- 行动规则 → principle

## 输出格式

追加到 `books/<slug>/candidates/insights.md`：

```yaml
- id: i01
  title: {{洞见标题}}
  type: insight
  source_chapter: ch004 / 第四章
  source_line: 42   # 引用在源章节文件中的行号（v3.22），用于阶段3直接定位
  source_quote: "{{≤150 字原文}}"
  summary: "作者提出的反直觉判断是什么；它颠覆了什么常识；为什么有迁移价值。"
  keywords: [洞见关键词, 常识反转]
  common_sense: "大多数人通常认为……"
  author_reversal: "作者真正指出的是……"
  v3_pass: true
  v3_reason: "读者没读过书很难自己说出这个反转。"
  v2_scenario: "可迁移到：{{新场景}}"
  related: [f02, p06]  # 交叉引用（v3.19）：关联的其他方法论单元 ID，可选但不超过3条
```

## 字段名强制规则（v3.19）

**输出字段名必须严格照此格式，不得更改。**
- ❌ 不要使用 `chapter:` 代替 `source_chapter:`
- ❌ 不要使用 `insight:` 代替 `summary:`
- ❌ 不要使用 `name:` 代替 `title:`
- ❌ 不要省略 `keywords:` 或 `v2_scenario:`
- ✅ 全部必需字段必须出现

## 引用层级校验（v3.14）

如果书籍是**解读型**（作者 X 解读思想家 Y 的思想），必须遵循：
- ✅ 只引用 **Y 的原话**作为 `source_quote`
- ❌ 不得引用 X 的解读、转述或评价作为 `source_quote`
- ⚠️ 如果找不到 Y 的原话佐证，标记 `v3_pass: false`，`v3_reason: "二次解读无原始引用"`

这条规则尤其适用于人物思想解读、投资大师语录整理、哲学家思想导读等类型的书籍。

## 自检

- [ ] 读起来不是「说得对」，而是「等等，和平常想的不一样」
- [ ] 写清楚常识与作者反转
- [ ] 引用 ≤150 字
- [ ] 没有把金句当洞见
