# Batch Processing Guidelines

> 批量处理多本书时的操作模式和已知阈值。v4.33 更新（API限流+源文件发现）。

## 操作模式

### 模式 A：全流程 split（推荐）

| 步骤 | 方式 | 耗时 | 成功率 |
|------|------|------|--------|
| 预处理+阶段1+阶段2 | `delegate_task` 并行3本 | 8-10min | ~70%（大书超时） |
| 阶段3 | `delegate_task` 并行3本 | 3-5min | ~95% |

**分开做的原因**：全流程 delegate_task 几乎 100% 在 600s 超时（预处理脚本 + execute_code 提取 + SUMMARY 写作总和超过时限）。拆分后各步骤在时限内。

### 模式 B：主代理直写阶段3（最快）

当候选数 ≤20 时，用 `write_file` 直接写 SUMMARY.md（跳过 delegate_task），2-3 分钟完成。适合小书。

### 模式 C：cron 批量消化（后台）

```python
cronjob(action='create', schedule='every 1h', skills=['jw-content-summary'],
  prompt='从 /root/books/ 挑下一本未处理的，执行全流程...')
```

每小时 1 本，2-3 天消化 50-70 本。跳过无源文件的、跳过已完成的。政治/马列类不再排除——用户明确要求全部处理。

## 已知阈值

| 书大小 | delegate_task 成功率 | 备注 |
|---------|---------------------|------|
| < 500KB | ~90% | 全流程或分步均可 |
| 500KB - 1MB | ~70% | 分步模式，全流程大概率超时 |
| 1MB - 2MB | ~50% | 预处理+阶段1+2 在边缘 |
| > 2MB | ~20% | 极易超时，建议分段或 cron |
| > 5MB | ~0% | 子代理无法完成，需主代理分章节处理 |

## ⚠️ API 限流应对（v4.33 新增）

**429 错误特征**：子代理运行到中途突然报 `HTTP 429: Token Plan 主要面向个人开发者的交互式使用场景`，3个并行任务全部被掐。

**防御策略**：
1. **连续批量时降并发**：不要同时起 3 个 subagent。等第一个完成再起第二个，或者只起 1 个并行任务。
2. **热状态下单本模式**：如果刚处理过几本，API 还热着，一次只处理 1 本，等 30-60s 再起下一个。
3. **感知 429 信号立即切换**：看到 429 马上下一个不要再发 API call，等 2 分钟再试。
4. **子代理内部不要堆叠 API**：阶段 2 execute_code 已经会消耗 token，子代理内再叠 execute_code 更容易触发阈值。

**实测数据**：
- 3 并行 → 10 分钟内必触 429（Token Plan 并发限制）
- 1 并行 → 每本平均 5-8 分钟，无 429

## ⚠️ 源文件发现是独立步骤（v4.33 新增）

**问题**：`/root/books/` 下的书有些是 `.txt` 源文件，有些是 `.md` 源文件（如 `通胀陷阱.md`）。子代理按 `.txt` 传参会报"文件不存在"，导致整本书跳过。

**防御策略**：每本书开始时第一步必须是文件发现：
```bash
ls /root/books/ | grep <书名关键词>   # 确认实际扩展名
```

**常见模式**：
- 大部分投资类书籍：`.txt`
- 部分整理过的书（可能有思维导图笔记）：`.md`
- `build_book_index.py` 两个扩展名都支持，无需特殊处理

**流程**：确认路径 → 传实际路径给 clean_text.py → 后续不变

## 空计划回退策略

`pipeline_phase2.py` 有时将所有候选打入 `excluded_or_weak`（全部为单成员 cluster 时常见）。此时：

1. `summary_plan.json` 的 `recommended_units` 和 `appendix_units` 均为空
2. **不重新提取**（重跑产生同质候选无意义）
3. 阶段 3 全量读取 `candidates/*.md` 所有候选
4. 手动编组为 10-16 个方法论主题组
5. 每个组选代表性候选展开 R/I/A1/A2/E/P/B 七段
6. 审计信息中记录空计划原因

## 验证清单

处理完每本书后：
- [ ] `SUMMARY.md` 存在且有内容（>5KB）
- [ ] `SUMMARY.html` 已生成
- [ ] 五问无占位符
- [ ] 至少 6 个方法论单元有完整七段
- [ ] 审计信息记录了 pipeline 决策
