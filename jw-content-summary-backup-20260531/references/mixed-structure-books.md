# 已知文件结构与处理模式

本文档记录各类书籍文本的已知结构异常和处理模式，供 stage1 分析时查阅。

## §7. 混合型书籍结构模式（v4.13新增）

### 识别特征

书名或副标题包含多个内容类型的组合：
- "Letters + 年会实录"（如《巴菲特致股东的信及伯克希尔股东大会实录》）
- "传记 + 投资框架"
- "案例集 + 方法论"
- "经济学讲义 + 现实分析"

### 识别方法

1. **cleaned_text 前100行扫描**：看是否有多种不同性质的 section 标题
2. **REGEX 定位标题行**：在 cleaned_text 中搜索关键标记（如 "巴菲特致股东的信\d{4}"、"股东大会实录\d{4}"）
3. **build_book_index 异常**：章节被切成 N 个 "Chunk N" 而非有意义的章节名 → 预示混合结构
4. **文件大小异常**：相比同类书（如《巴菲特之道》498KB），该书超大（12.9MB）→ 预示多内容合并

### 处理模式

**不要依赖** `build_book_index.py` 的通用切分（会切出几百个无意义的 Chunk）。

**正确做法**：
1. 在 cleaned_text 中用 REGEX 定位所有 section 标题行
2. 建立手工 section 映射表（start_line → end_line → type）
3. 按 section type 分配 extractor
4. 章节文件用自行切分的 txt 而非通用脚本的输出

### 典型案例

**《巴菲特致股东的信及伯克希尔股东大会实录》**
- 源文件：12.9MB / 123,919 行
- 结构：57封 Letters（1965-2024）+ 38个股东大会Q&A实录（1986-2025）
- 正确切分：用 REGEX `^巴菲特致股东的信(\d{4})` 和 `^伯克希尔股东大会实录(\d{4})` 定位标题行
- 总章节：95个（ch001-ch095）
- build_book_index 输出：949个 Chunk（无效）
- extractor 分配：principle/framework/insight/boundary 均适用，glossary 适用，case 从 Q&A 中提取
