# 内容分级处理详解

> 版本: v3.0.0
> 更新: 2026-05-30

## 概述

内容分级处理根据素材长度自动选择处理策略，优化 token 使用效率。不同长度的素材需要不同的处理深度：

- **长素材** (>1000字): 包含丰富信息，需要完整提取
- **中等素材** (300-1000字): 包含核心信息，需要标准处理
- **短素材** (<300字): 信息有限，简化处理即可

## 分级标准

| 级别 | 字符数 | 处理策略 | Token 预估 | 时间预估 |
|------|--------|----------|------------|----------|
| full | >1000 | 完整提取 | ~3000-5000 | 2-5分钟 |
| standard | 300-1000 | 标准提取 | ~1500-3000 | 1-2分钟 |
| simple | <300 | 简化提取 | ~500-1500 | 30秒-1分钟 |

## 处理策略详解

### full 级别（完整处理）

**适用场景：** 长篇文章、完整书籍章节、详细报告

**处理步骤：**
1. **完整提取所有实体和概念**
   - 识别所有人物、组织、产品、模型
   - 识别所有概念、主题、理论
   - 识别所有事件、日期、数据

2. **创建独立 wiki 页面**
   - 每个实体/概念创建独立页面
   - 包含完整 frontmatter
   - 包含详细内容和结构

3. **建立完整交叉引用**
   - 每页至少 2 个出站链接
   - 检查并更新反向链接
   - 建立双向引用关系

4. **标注置信度**
   - EXTRACTED: 原文直接支持
   - INFERRED: 从原文推断
   - AMBIGUOUS: 有歧义
   - UNVERIFIED: 来自背景知识

5. **添加来源标记**
   - 在相关段落末尾添加 `^[raw/...]`
   - 支持多来源追溯

6. **更新导航和日志**
   - 更新 index.md
   - 更新 log.md

**示例：**
```bash
# 处理一篇 2000 字的投资分析文章
bash content-grading.sh grade raw/articles/buffett-analysis-2026.md
# 输出: 处理级别: full
# 建议: 完整提取所有投资理念、人物关系、案例分析
```

### standard 级别（标准处理）

**适用场景：** 中等长度文章、摘要、评论

**处理步骤：**
1. **提取核心实体（3-5个）和关键概念**
   - 识别主要人物和组织
   - 识别核心概念和主题
   - 忽略次要细节

2. **创建或更新页面**
   - 创建新页面（如果实体不存在）
   - 更新已有页面（如果实体已存在）
   - 保持页面简洁

3. **建立基本交叉引用**
   - 每页 1-2 个出站链接
   - 链接到最相关的页面

4. **标注置信度**
   - 主要使用 INFERRED
   - 关键信息使用 EXTRACTED

5. **更新导航和日志**
   - 更新 index.md
   - 更新 log.md

**示例：**
```bash
# 处理一篇 500 字的投资评论
bash content-grading.sh grade raw/articles/market-commentary.md
# 输出: 处理级别: standard
# 建议: 提取核心观点和关键人物，创建简要页面
```

### simple 级别（简化处理）

**适用场景：** 短文、笔记、片段、引用

**处理步骤：**
1. **提取 1-2 个关键信息点**
   - 识别最重要的实体或概念
   - 忽略次要信息

2. **优先合并到已有页面**
   - 检查是否有相关页面
   - 将信息添加到已有页面
   - 避免创建新页面

3. **如需创建页面，保持简洁**
   - 页面内容 <50 行
   - 只包含核心信息
   - 简化 frontmatter

4. **简化交叉引用**
   - 只链接到最相关的 1 个页面
   - 或不添加链接

5. **更新导航和日志**
   - 更新 index.md
   - 更新 log.md

**示例：**
```bash
# 处理一段 200 字的笔记
bash content-grading.sh grade raw/notes/investment-tip.md
# 输出: 处理级别: simple
# 建议: 提取关键信息，合并到已有投资页面
```

## Token 效率优化

### 统一处理 vs 分级处理

假设处理 100 个素材文件：

**统一处理（全部按 full 级别）：**
- 100 文件 × 4000 tokens = 400,000 tokens

**分级处理：**
- 20 个 full × 4000 = 80,000
- 50 个 standard × 2000 = 100,000
- 30 个 simple × 1000 = 30,000
- 总计: 210,000 tokens

**节省:** 190,000 tokens (47.5%)

### 实际应用

```bash
# 统计目录内素材分级
bash content-grading.sh stats raw/articles/

# 输出示例：
# 文件总数: 100
# 总字符数: 150,000
# 
# 分级统计:
#   完整处理 (>1000字): 20 文件
#   标准处理 (300-1000字): 50 文件
#   简化处理 (<300字): 30 文件
# 
# === Token 效率 ===
# 统一处理预估: 400,000 tokens
# 分级处理预估: 210,000 tokens
# 节省: 190,000 tokens (47%)
```

## 最佳实践

### 1. 批量处理前先统计

```bash
# 处理前先统计
bash content-grading.sh stats raw/articles/

# 根据统计结果规划处理策略
# 如果大部分是 simple 级别，可以批量处理
# 如果大部分是 full 级别，需要分批处理
```

### 2. 结合缓存机制

```bash
# 先检查缓存
bash cache.sh check raw/articles/buffett-analysis.md

# 如果是 MISS，再判断处理级别
bash content-grading.sh grade raw/articles/buffett-analysis.md

# 根据级别处理
# 处理后更新缓存
bash cache.sh update raw/articles/buffett-analysis.md
```

### 3. 优先处理高质量素材

```bash
# 统计所有素材
bash content-grading.sh stats raw/

# 优先处理 full 级别的素材
# 这些素材信息最丰富，价值最高
```

### 4. 定期优化

```bash
# 每月统计一次
bash content-grading.sh stats raw/

# 根据统计结果调整分级阈值
# 如果大部分素材都是 simple，可能需要提高阈值
```

## 常见问题

### Q: 如何调整分级阈值？

A: 编辑 `scripts/content-grading.sh` 中的阈值变量：
```bash
THRESHOLD_FULL=1000     # >1000字：完整处理
THRESHOLD_MEDIUM=300    # 300-1000字：标准处理
# <300字：简化处理
```

### Q: 如何处理混合长度的素材？

A: 对于包含多种长度内容的素材：
1. 先判断整体长度
2. 如果整体是 full 级别，但包含短片段，可以：
   - 完整处理主体内容
   - 简化处理短片段
   - 或将短片段合并到相关页面

### Q: 如何确保质量？

A: 分级处理不影响质量，只是优化处理深度：
- **full**: 完整提取，质量最高
- **standard**: 核心提取，质量良好
- **simple**: 关键提取，质量基本

### Q: 如何处理重要但短小的素材？

A: 对于重要但短小的素材：
1. 手动判断为 standard 或 full 级别
2. 使用 `bash content-grading.sh grade` 查看建议
3. 根据实际价值调整处理策略
