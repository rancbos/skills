# 别名系统详解

## 概述

别名系统用于定义和管理同义词，提高搜索命中率。当用户搜索某个关键词时，自动展开为所有同义词进行搜索，避免因术语不同而遗漏相关内容。

**核心价值：** 同一个概念可能有多种表述方式，别名系统确保无论用户使用哪种表述，都能找到相关内容。

## 别名文件

**位置：** `$WIKI_PATH/.wiki-aliases.json`

**格式：**
```json
{
  "别名组1": ["词1", "词2", "词3"],
  "别名组2": ["词A", "词B"]
}
```

**示例：**
```json
{
  "儒家": ["儒家", "儒学", "孔孟", "儒教"],
  "道家": ["道家", "道学", "老庄", "道教"],
  "法家": ["法家", "法家学派", "韩非子", "商鞅"],
  "兵家": ["兵家", "兵法", "孙子", "孙武"],
  "价值投资": ["价值投资", "巴菲特", "芒格", "格雷厄姆"]
}
```

## 使用方法

### 1. 列出所有别名组

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh list
```

**输出示例：**
```
=== 别名组列表 ===
  - 儒家
  - 道家
  - 法家
  - 兵家
  - 价值投资
```

### 2. 获取关键词的别名组

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh get "儒家"
```

**输出示例：**
```
儒家
```

### 3. 添加别名组

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh add "儒家" "儒家" "儒学" "孔孟" "儒教"
```

**参数说明：**
- 第一个参数：组名
- 后续参数：该组的所有同义词

### 4. 展开关键词的所有别名

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh expand "儒家"
```

**输出示例：**
```
儒学 儒家 儒教 孔孟
```

### 5. 搜索时使用别名展开

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh search "儒家"
```

**输出示例：**
```
搜索关键词：儒家
展开后：儒学 儒家 儒教 孔孟

  '儒学'：找到 5 个页面
  '儒家'：找到 8 个页面
  '儒教'：找到 3 个页面
  '孔孟'：找到 2 个页面
```

## 在查询工作流中的使用

### 查询流程（含别名展开）

```
用户查询关键词
    ↓
读取 .wiki-aliases.json
    ↓
查找包含该关键词的别名组
    ↓
├─ 找到 → 展开为所有同义词
└─ 未找到 → 使用原始关键词
    ↓
搜索所有展开的关键词
    ↓
综合回答并引用 wiki 页面
```

### 示例

**用户查询：** "儒家的核心思想是什么？"

**别名展开：**
- 关键词"儒家"命中别名组"儒家"
- 展开为：["儒家", "儒学", "孔孟", "儒教"]

**搜索结果：**
- 搜索"儒家"：找到 8 个页面
- 搜索"儒学"：找到 5 个页面
- 搜索"孔孟"：找到 2 个页面
- 搜索"儒教"：找到 3 个页面

**综合回答：** 基于所有找到的页面，综合回答用户问题。

## 别名组设计原则

### 1. 同一概念的不同表述

```json
{
  "LLM": ["LLM", "大语言模型", "大模型", "Large Language Model"]
}
```

**原因：** 同一个概念可能有多种表述方式，别名系统确保都能搜索到。

### 2. 相关人物的别名

```json
{
  "价值投资": ["价值投资", "巴菲特", "芒格", "格雷厄姆"]
}
```

**原因：** 这些人物都与价值投资相关，搜索"价值投资"时应该也找到他们的相关内容。

### 3. 学派的别名

```json
{
  "儒家": ["儒家", "儒学", "孔孟", "儒教"],
  "道家": ["道家", "道学", "老庄", "道教"]
}
```

**原因：** 学派有多种称呼方式，别名系统确保都能搜索到。

### 4. 避免过度泛化

**错误示例：**
```json
{
  "投资": ["投资", "股票", "基金", "理财", "财富"]
}
```

**问题：** 这些词含义不同，不应该放在同一别名组。

**正确示例：**
```json
{
  "投资": ["投资", "理财"],
  "股票": ["股票", "股市"],
  "基金": ["基金", "公募", "私募"]
}
```

## 与 llm-wiki 的对比

| 特性 | llm-wiki | jw-llm-wiki |
|------|----------|-------------|
| 别名文件 | .wiki-schema.md 中的"别名词表" | .wiki-aliases.json |
| 展开规则 | 只在命中的那一组内展开，不跨组传递 | 只在命中的那一组内展开，不跨组传递 |
| 去重 | 自动去重 | 自动去重 |
| 搜索优化 | 同一别名组的多个词命中同一页面时只计一次 | 同一别名组的多个词命中同一页面时只计一次 |

## 最佳实践

### 1. 定期维护别名

随着知识库的增长，可能需要添加新的别名组。定期检查和维护别名词表。

### 2. 避免别名冲突

确保不同别名组之间没有重叠的词。如果一个词属于多个组，应该合并或重新组织。

### 3. 测试别名展开

添加新别名组后，测试展开功能是否正常工作：

```bash
bash ${SKILL_DIR}/scripts/alias-manager.sh expand "新关键词"
```

### 4. 文档化别名

在 wiki 的 schema 或相关文档中记录别名组的设计意图，方便后续维护。

## 故障排除

### 问题：别名展开不工作

**原因：** 别名文件格式错误

**解决：**
```bash
# 检查 JSON 格式
cat $WIKI_PATH/.wiki-aliases.json | python3 -m json.tool
```

### 问题：搜索结果不准确

**原因：** 别名组设计不合理

**解决：**
```bash
# 查看当前别名组
bash ${SKILL_DIR}/scripts/alias-manager.sh list

# 删除不合理的别名组
# （需要手动编辑 .wiki-aliases.json）
```

### 问题：别名文件丢失

**原因：** 文件被误删

**解决：**
```bash
# 重新创建别名文件
echo "{}" > $WIKI_PATH/.wiki-aliases.json

# 重新添加别名组
bash ${SKILL_DIR}/scripts/alias-manager.sh add "儒家" "儒家" "儒学" "孔孟" "儒教"
```
