# 结晶化工作流详解

> 版本: v3.0.0
> 更新: 2026-05-30

## 概述

结晶化工作流用于捕获对话中的洞见，将其转化为 wiki 页面。这是一个从临时想法到持久知识的转化过程。

## 工作流程

```
对话中发现洞见
    ↓
保存到 _crystallize/pending.md
    ↓
定期查看待处理洞见
    ↓
判断是否值得创建 wiki 页面
    ↓
是 → 创建页面 → 更新 index.md → 移动到 processed.md
否 → 标记为已处理 → 移动到 processed.md
```

## 使用方法

### 保存洞见

```bash
# 保存洞见
bash crystallize.sh save "价值投资的核心是安全边际，而不是预测市场" "对话中的讨论"

# 输出示例：
# ✅ 洞见已保存
# 标题: 价值投资的核心是安全边际...
# 来源: 对话中的讨论
# 位置: /root/wiki-ai/_crystallize/pending.md
```

### 列出待处理洞见


### 处理洞见



## 判断标准

### 值得创建 wiki 页面的洞见

1. **原创性**: 洞见包含新的观点或分析
2. **深度**: 洞见有详细的论证或案例支持
3. **关联性**: 洞见与已有知识有联系
4. **持久性**: 洞见在未来仍然有价值
5. **可验证性**: 洞见可以被验证或引用

**示例：**
- ✅ "价值投资的核心是安全边际，而不是预测市场"
  - 原创性: 高（核心观点）
  - 深度: 高（可以展开论述）
  - 关联性: 高（与多个投资概念相关）
  - 持久性: 高（长期有效）
  - 可验证性: 高（可以引用巴菲特）

- ❌ "今天市场涨了"
  - 原创性: 低（事实陈述）
  - 深度: 低（无分析）
  - 关联性: 低（与知识体系无关）
  - 持久性: 低（很快过时）
  - 可验证性: 高（但无意义）

### 不值得创建 wiki 页面的洞见

1. **琐碎信息**: 简单的事实陈述
2. **临时想法**: 没有深入思考的想法
3. **重复内容**: 已经在 wiki 中存在的内容
4. **领域外**: 不属于 wiki 覆盖的领域
5. **无法验证**: 无法被验证或引用

## 最佳实践

### 1. 及时保存洞见

```bash
# 对话中发现洞见时立即保存
bash crystallize.sh save "洞见内容" "来源描述"

# 不要等到对话结束再保存
# 可能会忘记或丢失
```

### 2. 定期处理洞见

```bash
# 每周处理一次洞见
bash crystallize.sh list

# 查看待处理洞见
# 判断哪些值得创建 wiki 页面
```

### 3. 保持洞见质量

```bash
# 保存洞见时提供详细信息
bash crystallize.sh save "详细的洞见内容" "具体的来源描述"

# 避免保存琐碎信息
# 只保存有价值的洞见
```

### 4. 与 wiki 工作流集成

```bash
# 处理洞见时
# 1. 先搜索 wiki 中是否已有相关内容
search_files "洞见关键词" path="$WIKI_PATH"

# 2. 如果已有，更新已有页面
# 3. 如果没有，创建新页面

# 4. 更新 index.md
# 5. 标记洞见为已处理
bash crystallize.sh mark "洞见标题"
```

## 常见问题

### Q: 如何处理大量洞见？

A: 对于大量洞见：
1. **优先处理高质量洞见**: 先处理最有价值的洞见
2. **批量处理**: 一次性处理多个相关洞见
3. **归档低质量洞见**: 将不值得处理的洞见直接标记为已处理

### Q: 如何避免重复洞见？

A: 保存前先搜索：
```bash
# 搜索已有洞见
grep "洞见关键词" _crystallize/pending.md
grep "洞见关键词" _crystallize/processed.md

# 搜索 wiki 中的相关内容
search_files "洞见关键词" path="$WIKI_PATH"
```

### Q: 如何提高转化率？

A: 提高转化率的方法：
1. **及时保存**: 对话中立即保存洞见
2. **详细描述**: 提供详细的洞见内容和来源
3. **定期处理**: 每周处理一次洞见
4. **质量筛选**: 只保存有价值的洞见

### Q: 如何处理跨领域的洞见？

A: 对于跨领域的洞见：
1. **识别核心领域**: 确定洞见属于哪个领域
2. **创建关联页面**: 创建多个相关页面
3. **建立交叉引用**: 在不同领域的页面间建立链接


### 2. 洞见关联

```bash
# 保存关联洞见

# 处理时创建关联页面
# entities/invest/safety-margin.md
# entities/invest/risk-control.md
# 建立交叉引用
```

### 3. 洞见验证

```bash
# 保存需要验证的洞见
bash crystallize.sh save "某个投资策略的历史表现" "需要验证"

# 处理时验证
# 1. 搜索相关数据
# 2. 验证洞见的准确性
# 3. 如果准确，创建页面
# 4. 如果不准确，标记为已处理并说明原因
```

### 4. 洞见扩展

```bash
# 保存基础洞见

# 处理时扩展
# 1. 创建基础页面
# 2. 添加详细内容

# 4. 更新 index.md
```
