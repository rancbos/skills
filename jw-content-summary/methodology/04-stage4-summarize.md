# 阶段 3 — 单文件 SUMMARY 渲染

## 目标

在不压缩最终质量的前提下，用 `clusters.json` 减少重复阅读，生成一份完整 `SUMMARY.md`。

## 输入

- `stage1-understanding.md`
- `summary_plan.json`（v4.4 首选输入，由 `clusters.json` + `candidate_scores.json` 生成）
- `clusters.json`
- `candidate_scores.json`
- `candidates/*.md`
- 必要原文片段（只为补足引用/案例/边界）

## 输出

- `books/<slug>/SUMMARY.md`

## SUMMARY.md 结构

```
# {{BOOK_TITLE}} — 方法论总结

## 一、作者的写作意图          ← 阶段3 步骤1：从 stage1-understanding.md 提取
## 二、这本书在谈什么          ← 阶段3 步骤1：从 stage1-understanding.md 提取
## 三、全书的内容结构          ← 阶段3 步骤1：从 book-index.json 结构生成
## 四、这本书有道理吗          ← 阶段3 步骤2：从 stage1-understanding.md 批判性评价提取
## 五、这本书与自己的关系      ← 阶段3 步骤2：从 stage1-understanding.md 提取
## 六、核心方法论              ← 阶段3 步骤3：读取 summary_plan.json + candidates，写 R/I/A1/A2/E/P/B 七段
## 七、压力测试                ← 阶段3 步骤4：对核心论点做 Musk/Socrates/Munger + 推理链审查
## 八、关联方法论              ← 阶段3 步骤5：本书内部关联 + 跨书关联 + 拓扑检验
## 附录：未完整展开的方法论    ← 阶段3 步骤6：summary_plan.json 的 appendix_units
## 审计信息                    ← 阶段3 步骤7：记录 pipeline 数据、版本、回退原因
```

### 默认规则

- 优先读取 `summary_plan.json`：
  - `recommended_units`：默认完整 R/I/A1/A2/E/P/B 七段展开
  - `appendix_units`：默认进入附录，除非用户要求完整展开
  - `excluded_or_weak`：默认不写入正文，除非返回阶段 2 补证或用户明确要求
- 如果缺少 `summary_plan.json`，回退到旧规则：cluster 数 ≤10 全部展开；cluster 数 >10 全量展开（v4.22 闸口已移除，全量模式）。
- `candidate_scores.json` 的 A/B/C/D 是决策信号，不是自动删除规则。

## RIAAEPB 七段

### R — Reading（原文）

- 直接引用 ≤150 字
- 必须标注出处（章节 / 页码 / 段落标识）
- 若原书是英文，引用英文原文 + 自己翻译的中文，**不要用现成译本**

### I — Interpretation（自述）

- 用自己的话重写方法论核心骨架
- 5–15 行
- 一个没读过原书的人能否理解？不能就重写

**费曼检验（v4.54 新增）**：I 段必须通过以下三项：
1. **术语去魅**：核心术语是否已"翻译"为日常语言？如果删掉所有专业术语，外行能否理解？
2. **比喻锚点**：复杂概念是否有恰当的日常比喻？（如"安全边际=买保险"、"复利=滚雪球"）
3. **逻辑完整**：论证链是否有跳跃？从 A 到 B 的推理是否显式？

### A1 — Past Application（书中案例）

- 作者在书中亲自用这个方法论处理过的具体案例
- 至少 1 条，≤3 条
- 问题 → 使用 → 结论 → 结果

**案例保真规则（v4.55 新增）**：

A1 段涉及案例、研究、数据时，必须遵守保真铁律：

| 场景 | 规则 |
|------|------|
| **有原文在手** | 禁止概括性改写；必须保留**具体数字、人名/机构、时间线、原话**（可标页码/章节） |
| **无原文、仅摘要或记忆** | 标注 `来源: 本书/摘要，未核对原文`；保留能确定的专有名词与结论，**禁止编造细节** |
| **英文原书** | 引用英文原文 + 自己翻译的中文，**不要用现成译本** |

**好案例 vs 坏案例**：
- ❌ 坏："巴菲特投资可口可乐赚了很多钱"（概括性改写，丢失关键数据）
- ✅ 好："1988年巴菲特投入6亿美元买入可口可乐，到1998年市值130亿，回报21倍"（保留数字和时间线）
- ❌ 坏："研究表明这个方法有效"（无来源、无数据）
- ✅ 好："Kahneman & Tversky (1979) 前景理论实验：200名受试者，87%在损失情境下选择冒险"（保留作者、年份、样本量、比例）

### A2 — Application Scenarios（应用场景）

- 3–5 条具体场景
- 可参考候选中的 `v2_scenario`
- 必须区分相邻方法论，避免混淆

### E — Key Points（执行要点）

- 1-2-3 要点
- 每步有可判断的完成标准
- 判停点显式写出

### P — Procedure（步骤）

P 不是 E 的重复。E 是「要做什么」，P 是「怎么做」。

P 段必须：
1. 还原因果链
2. 区分 🔵必经 与 🟡条件触发
3. 显式化隐性前提
4. 用可操作动词，不用原则替代步骤
5. 通过新手反向验证

**模糊词禁令（v4.55 新增）**：

P 段禁止出现以下模糊词，必须替换为具体动作或量化指标：

| 禁止的模糊词 | 必须替换为 |
|-------------|-----------|
| 优化 | 具体优化什么、怎么优化（如"将响应时间从200ms降到50ms"） |
| 加强 | 加强什么、加到什么程度（如"每天练习30分钟"） |
| 适当 | 具体多少（如"仓位不超过总资产的10%"） |
| 合理 | 合理的标准是什么（如"PE低于行业均值20%"） |
| 有效 | 什么算有效、怎么衡量（如"胜率>60%"） |
| 关注 | 具体看什么指标、什么频率（如"每天检查一次止损位"） |
| 注意 | 注意什么、不注意会怎样（如"必须在收盘前平仓，否则隔夜风险"） |
| 尽量 | 🔵必经还是🟡条件？必须明确 |
| 可能 | 到底可能还是不可能？概率多少？ |
| 相关 | 什么关系？因果还是相关？方向是什么？ |

**好P段 vs 坏P段**：
- ❌ 坏："适当优化仓位管理"（3个模糊词）
- ✅ 好："将单笔仓位从20%降到10%，总仓位不超过60%，每次加仓间隔≥3天"（全部量化）

**P 段增强（v4.54 新增）**：在步骤之后，补充两个子节：

**机制与杠杆**：
- 这个方法为什么有效？底层机制是什么？
- 哪 20% 的步骤决定了 80% 的效果？（帕累托杠杆）
- 隐含假设是什么？（如：假设市场是理性的、假设用户有基础认知）

**最小可行性实验（MVE）**：
- 本周内可以执行的最小实验是什么？
- 具体行动：1-2 个可立即执行的动作
- 预期结果：怎么判断这个方法对我有效？

### B — Boundary（边界）

- 什么时候不要用
- 作者警告过的失败模式
- 容易混淆的相邻方法论

## 七、压力测试（v4.54 新增）

> 来源：deep-learning skill 的 Critics 座席。对全书核心论点做压力测试，暴露盲区。

在五问之后、核心方法论之前，对全书核心论点做三视角压力测试：

### Musk 视角（第一性原理质疑）
- 作者的核心假设是否成立？从物理/数学/经济学第一性原理看，有没有根本性矛盾？
- 如果这个假设是错的，整个论证会怎样崩塌？

### Socrates 视角（概念诘问）
- 核心概念的定义是否精确？有没有循环定义或偷换概念？
- 作者是否在不同章节对同一概念用了不同含义？

### Munger 视角（逆向思考）
- 反过来想：如果作者的结论是错的，会怎样？
- 有什么潜在风险/反效应/二阶后果作者没有讨论？
- 在什么极端条件下这个方法论会失效？

### 推理链审查（v4.55 新增）
> 来源：deep-learning skill 的缺口分析（Gap Analysis）。识别论证链中的跳跃。

列出作者从前提到结论的完整推理链，标记每一步的证据强度：

```markdown
推理链：前提A → [证据强度] → 中间结论B → [证据强度] → 最终结论C

*   前提A：[作者的核心假设] → 🟢强/🟡弱/🔴缺失
*   A→B：[推理步骤] → 🟢有数据支撑/🟡逻辑合理但无数据/🔴跳跃
*   B→C：[推理步骤] → 🟢有数据支撑/🟡逻辑合理但无数据/🔴跳跃
*   结论C：[作者的最终结论]
```

**关键问题**：
- 作者从 A 跳到 C，中间的 B 是否有论证？
- 哪些步骤是"逻辑合理但无数据"（合理推断）？
- 哪些步骤是"直接跳跃"（论证缺口）？

**注意**：论证缺口不等于结论错误——有些推断是合理的，只是作者没有显式论证。标注缺口是为了提醒读者需要自行验证。

**输出格式**：
```markdown
## 七、压力测试

*   **Musk**：[第一性原理质疑]
*   **Socrates**：[概念定义诘问]
*   **Munger**：[逆向思考/潜在风险]
*   **推理链**：前提A → 🟢/🟡/🔴 → 中间结论B → 🟢/🟡/🔴 → 最终结论C
*   **结论**：[辩论后的共识或存疑点；标注哪些论证缺口需要读者自行验证]
```

**注意**：压力测试不是否定作者，而是检验论证的鲁棒性。如果三个视角都找不到致命漏洞，说明这本书的方法论经得起推敲。

## 八、关联方法论（v4.54 新增）

> 来源：deep-learning skill 的 Luhmann 知识连接。让方法论不再孤立。

在核心方法论之后，建立本书方法论与已有知识网络的连接：

### 与本书其他方法论的关系
- 本书内部各方法论之间的依赖/互补/冲突关系
- 哪些方法论必须组合使用？哪些互斥？

### 与已读过书籍的关联
- 与之前读过的哪些书的方法论相似/对立/互补？
- 具体到方法论级别：如"本书的X方法与《聪明的投资者》的安全边际是同一思路"
- **已读书籍路径**：默认在 `<源文件.txt所在目录>/` 下搜索同级书名目录中的 `SUMMARY.md`（glob pattern: `*/SUMMARY.md`）；也可在 `books/` 下按书名关键词搜索（`books/*书名*/SUMMARY.md`）。如果没有找到已读书籍的 SUMMARY，只写本书内部关联和跨领域连接。

### 跨领域连接
- 这个方法论在其他领域有没有对应物？（如经济学的"边际效用"≈心理学的"快乐跑步机效应"）

### 拓扑检验（v4.55 新增）
> 来源：deep-learning skill 的 Feynman Review 第四项——"意外的惊喜连接"。

在梳理关联时，主动寻找**意外连接**——两个看似无关的方法论，是否共享同一个底层机制？

**检查清单**：
- 本书内部：有没有两个方法论表面不同、但底层机制相同？（如"安全边际"和"能力圈"都基于"信息不对称"）
- 跨书：有没有不同书的方法论其实是同一思维模型的不同表述？（如芒格的"反过来想"≈军事的"蓝军思维"≈保险的"逆向选择"）
- 跨领域：有没有在完全不同的领域发现了相同的模式？（如物理学的"熵增"≈组织管理的"大企业病"≈投资的"均值回归"）

**输出格式**：
```markdown
### 拓扑检验
- 🎯 意外连接：[方法论A] 和 [方法论B] 看似无关，但共享底层机制 [X]
- 🎯 同构映射：[领域1] 的 [概念] ≈ [领域2] 的 [概念]（同一个模式的不同表达）
```

**注意**：拓扑检验不是必须有结果——如果梳理后确实没有意外连接，写"未发现意外连接"即可。但**必须主动寻找**，不能跳过。

## 方法论单元阅读顺序（v4.55 新增）

> 来源：deep-learning skill 的结构笔记模板——"Reading Sequence"。

在"三、全书的内容结构"中，如果方法论单元之间存在前置依赖关系，必须标注**推荐阅读顺序**：

**触发条件**：当满足以下任一时，必须输出阅读顺序：
1. 方法论 A 的 I 段引用了方法论 B 的概念（B 是 A 的前置）
2. 方法论 C 的 P 段依赖方法论 D 的输出（D 必须先执行）
3. 作者在书中显式说"在理解 X 之前，先理解 Y"

**输出格式**（在"三、全书的内容结构"末尾追加）：
```markdown
### 推荐阅读顺序

#### 路径一：[主题名]
1.  **方法论X**（前置基础）
    > 理由：Y 的核心概念"Z"依赖 X 的定义
2.  **方法论Y**（核心应用）
    > 理由：Y 在 X 的基础上构建操作步骤

#### 路径二：[另一个主题]
1.  **方法论A**
    > 理由：独立主题，无前置依赖
```

**注意**：如果所有方法论单元之间没有显式依赖关系，不需要输出阅读顺序——直接按 cluster 聚合顺序即可。不要为没有依赖的方法论强行排序。

**输出格式**：
```markdown
## 八、关联方法论

### 本书内部关联
- 方法论A ↔ 方法论B：互补关系（A定义边界，B提供操作步骤）
- 方法论C ↔ 方法论D：互斥关系（C适用于牛市，D适用于熊市）

### 跨书关联
- 本书"安全边际" ↔ 《聪明的投资者》"安全边际"：相同概念，本书更强调量化
- 本书"逆向投资" ↔ 《穷查理宝典》"反过来想"：同一思维模型的不同表述

### 跨领域连接
- 本书"复利效应" ↔ 数学"指数增长" ↔ 心理学"延迟满足"
```

**注意**：关联方法论需要用户有已读书籍的 SUMMARY.md 作为输入。如果没有，只写本书内部关联和跨领域连接。

## 常见失败模式

1. **cluster 不处理** — 同一方法论重复展开多次
2. **P 段变成 E 段复读** — P 要有输入/操作/输出链
3. **独特洞见写成赞美** — "这本书很深刻"不是洞见
