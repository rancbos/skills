# validate_summary.py 格式要求

`validate_summary.py` 使用严格正则匹配验证 SUMMARY.md。以下格式要求必须满足，否则校验失败。

## 两个最高频失败点（优先检查）

| # | 失败现象 | 根因 | 修复 |
|---|---|---|---|
| 1 | `Audit section not found` | 正文中没有字面量 `审计信息`（只有 `审计区` 等） | 写 `审计信息：` 标题，或正文含该词；若无 candidates 目录则自动跳过 |
| 2 | `[MISSING] No R-section chapter citations found` | R段引用超150字，或章节引用格式错误 | 截断R段引用；章节引用用 `> ——第N章`、`> ——第X篇`、`> ——《书名》，chXXX` 或 `> ——《书名》，作者名` |

## ⚠️ 章节引用格式关键陷阱（v4.46 新增）

**`> —— chXXX` 格式不生效。** validate_summary.py 对 `chXXX` 格式的检测正则要求书名必须在 `《》` 中：

```python
r'> ——\s*《[^》]+》，ch[0-9]+'   # 必须有书名
```

单独 `> —— ch241` 不会被检测为章节引用，validate 报 `[MISSING] No R-section chapter citations found`。

**正确格式**：`> ——《书名》，chXXX`（如 `> ——《巴菲特致股东的信及伯克希尔股东大会实录》，ch241`）

**⚠️ 副标题/卷号必须在《》内部（v4.49 新增陷阱）**：

当书名含副标题或卷号时，副标题/卷号必须放在 `《》` 括号**内部**，不能在外面。因为正则 `《[^》]+》` 匹配到 `》` 就停止，之后期望紧跟 `，`，如果中间有副标题文字则匹配失败。

| 格式 | 示例 | 结果 |
|---|---|---|
| ❌ 卷号在括号外 | `> ——《习近平谈治国理政》第四卷，ch003` | FAIL — `[^》]+` 匹配到 `》` 后期望 `，`，但遇到 `第` |
| ✅ 卷号在括号内 | `> ——《习近平谈治国理政·第四卷》，ch003` | PASS |
| ✅ 副标题在括号内 | `> ——《原则——应对变化中的世界秩序》，ch007` | PASS |

**批量修复命令**：
```bash
# 将 《书名》副标题，chXXX 改为 《书名·副标题》，chXXX
sed -i 's/> ——《\([^》]*\)》\([^，]*\)，ch/> ——《\1·\2》，ch/g' SUMMARY.md
```

**批量修复命令**（当多个章节引用需要统一替换时）：
```bash
sed -i 's/> —— chXXX/> ——《书名》，chXXX/g' SUMMARY.md
# 例：将所有 > —— ch241 替换为 > ——《书名》，ch241
```

这在从纯文本书籍（无书名信息）逆向推导 SUMMARY 时尤为重要——阶段1/2的chunks文件只含 `chXXX`，没有书名上下文，必须在交付前补充书名。

## R-section 格式（v4.33 修复版）

实际正则（`scripts/validate_summary.py`）：
```python
r'\*\*R — Reading[^\n]*\n\n> (.+?)\n> ——'
```

**正确格式**：
```
**R — Reading（原文引用）**：

> 直接引用的原文内容，不超过150字...

> ——第4章
```

关键点：
- 节标题必须是 `**R — Reading**：`（带英文标签，双星号加粗）
- 标题后必须空一行（`\n\n`）
- 引用以 `> ` 开头（blockquote）
- 章节出处必须在**单独一行**，以 `> ——第X章` 格式（不是 `（第X章）`）
- `> ——` 必须在独立行上，与引用内容用空行分隔
- 正则支持：标题中含中文括号 `（原文引用）`、可选闭合 `**`、全角冒号 `：**`

**错误格式（校验不通过）**：
```
**R — Reading（原文引用）**：> "引文..."
> ——第4章
  ← `> ——` 必须独立成行，不能与引文同在一行
```

## 章节引用格式（v4.40 三格式并列）

支持三种格式并列检测，任一出现即通过：

| 格式 | 正则 | 示例 |
|---|---|---|
| 中文章节 | `r'> ——\s*第[零一二三四五六七八九十0-9篇章节]+'` | `> ——第7章`、`> ——第七章`、`> ——序二`、`> ——第5篇` |
| `chXXX` 格式 | `r'> ——\s*《[^》]+》，ch[0-9]+'`（re.IGNORECASE） | `> ——《周金涛理论大集》，ch008` |
| 作者名格式 | `r'> ——\s*《[^》]+》，[^，,]+$'` | `> ——《原则》，瑞·达利欧` |

**注意**：
- `chXXX` 格式要求书名在 `《》` 中，来源标识必须以 `ch` 开头后跟数字
- **`篇` 字是有效的章节标记**（2026-05-29 验证）：讲话/文稿合集无章节标题时，使用 `> ——第X篇` 格式可通过 validator。正则中的 `篇章节` 三字是 OR 关系
- **年份+信格式需加"第"字**（2026-05-31 验证）：信件合集（如巴菲特致股东的信）常用 `> ——1965年1月18日信` 格式，但 validator 的中文章节正则 `第[零一二三四五六七八九十0-9篇章节]+` 要求必须有"第"字。`> ——1965年信` 不通过，`> ——第1965年信` 通过。批量修复：`sed -i 's/> ——\([0-9]\{4\}\)年/> ——第\1年/g' SUMMARY.md`

## H2 段落标题变体（v4.39 支持）

部分书籍（如《原则——应对变化中的世界秩序》逆向推导）使用 H2 标题而非 H3 bold：

```markdown
## R段：核心引文（Reason/引用）

## I段：内容简介（Introduction）
```

section completeness 检查使用**双前缀匹配**：
- H2 前缀：`['R段', 'I段', 'A1段', 'A2段', 'E段', 'P段', 'B段']`
- H3 前缀：`['R —', 'I —', 'A1 —', 'A2 —', 'E —', 'P —', 'B —']`

任意前缀匹配即视为该段存在。

## H4 标题变体（v4.51 文档化）

部分书籍使用 H4 标题（`#### R段（原文引用）`）而非 H3 粗体（`**R — Reading**：`）。

**校验覆盖差异**：

| 检测 | `#### R段`（H4 标题） | `**R — Reading**：`（H3 粗体） |
|------|:---:|:---:|
| Section completeness（七段齐全） | ✅ 通过（`R段` 匹配 H2 前缀） | ✅ 通过（`R —` 匹配 H3 前缀） |
| R-block quote length（≤150字符） | ❌ **被跳过** | ✅ 检查 |

**根因**：R-block 长度检查正则 `r'\*\*R — Reading[^\n]*\n\n> (.+?)\n> ——'` 只匹配 `**R — Reading` 粗体格式。如果只用 H4 标题，该正则找不到匹配，R-block 长度检查完全不执行，引文即使超 150 字符也不会被拦截。

**推荐安全格式**（H4 标题 + 内联粗体行，确保两套检测都覆盖）：
```markdown
#### R段（原文引用）

**R — Reading**：

> 引文内容（≤150字符）

> ——第X章
```

**不安全格式**（R-block 长度检查被跳过）：
```markdown
#### R段（原文引用）
> 引文内容（可能超150字符但不会被检测）

> ——第X章
```

## 节标题格式

所有七段节标题必须匹配以下模式（check completeness 用前缀匹配）：

| 段 | H3 正确格式 | H2 变体格式 |
|---|---|---|
| R | `**R — Reading**：` | `## R段：...` |
| I | `**I — Interpretation**：` | `## I段：...` |
| A1 | `**A1 — Past Application**：` | `## A1段：...` |
| A2 | `**A2 — Application Scenarios**：` | `## A2段：...` |
| E | `**E — Key Points**：` | `## E段：...` |
| P | `**P — Procedure**：` | `## P段：...` |
| B | `**B — Boundary**：` | `## B段：...` |

## 五问格式（v4.40 三格式并列）

validate 支持三种五问格式，**任一完整即通过**：

| 格式 | 示例 | 检测关键词 |
|---|---|---|
| 标准五问 | `作者的写作意图` / `这本书在谈什么` | 5个标准短语（作者的写作意图、这本书在谈什么、全书的内容结构、这本书有道理吗、这本书与自己的关系） |
| 编号五问 | `### 1. 这本书解决什么问题？` 等 | `这本书解决什么问题`、`作者的核心观点是什么`、`这本书的回答有什么独特之处`、`普通人最应该记住什么`、`这本书适用于哪些场景` |
| Q格式 | `### Q1：这个理论体系的核心理念是什么？` 等 | `Q1：`、`Q2：`、`Q3：`、`Q4：`、`Q5：` |

**通过条件**：标准五问≥5 OR 编号五问≥5 OR Q格式≥5。完全缺失任一五问才报错。

## 审计区检查（v4.39 条件跳过）

`审计信息` 必须出现在文中（`in` 匹配）。

**自动跳过**：若书籍无 candidates 目录（逆向推导书或无阶段2产物），审计节检查自动跳过，不报错。

## 常见失败模式与修复

### 1. `[MISSING] No R-section chapter citations found`
原因：章节引用格式不对（如 `（第4章）` 而非 `> ——第4章`），或用了 `chXXX` 但书名不在 `《》` 中  
修复：用 Python regex 做上下文感知修复：
```python
import re
# 支持三种格式
content = re.sub(r'（第(\d+)章）', r'\n> ——第\1章', content)  # 中文括号 → 独立行
content = re.sub(r'> ——\s*《([^》]+》，ch(\d+)', r'> ——《\1》，ch\2', content)  # 规范化
```

### 2. `[MISSING] Unit N lacks section: R — Reading`
原因：节标题格式不匹配（如 `#### R段` 而非 `**R — Reading**：` 或 `## R段：`）  
修复：统一七段标题格式，优先用 H3 bold 格式

### 3. 五问全部 MISSING
原因：使用了非标准五问表述（如 `### Q1：...` 而非 `### 1. ...` 且不在标准五问列表中）  
修复：确保五问中至少包含标准五问的5个关键词，或使用 `### 1. 这本书解决什么问题？` 等编号格式

### 4. sed 陷阱
用 sed 批量修复章节引用时，容易：
- 匹配到非 R-section 的章节引用（如内容描述中的"（第4章）"）
- 产生 `> text...> ——第X章`（同一行，非期望的多行格式）

**推荐用 Python regex** 做上下文感知的修复，而非 sed。

### 5. R-section 格式损坏导致的级联超长（v4.47 新增）

**触发场景**：《温铁军套装》Unit 11，R-section 标题与引文之间多了一个空行，导致 validator 的 R-section 正则 `\n> (.+?)\n> ——`（DOTALL）在缺少正确的 `> ——` 分离行时，`.+?` 跨 block 匹配，一直匹配到 Unit 12 的 I-section 开头。

**级联效应**：
1. Unit N 的 R-block 捕获了下一个 unit 的 I-section（1232 chars），远超 150 限制
2. 后续 units 序号全部错位，一个超长 block 导致 validator 报多个 unit 超限
3. 修复时若只改前几个 unit，后续 unit 的 R/I 结构仍然错乱

**判断信号**：
```python
import re
with open('SUMMARY.md', 'r') as f:
    content = f.read()
r_blocks = re.findall(r'\*\*R — Reading[^\n]*\n\n> (.+?)\n> ——', content, re.DOTALL)
for i, block in enumerate(r_blocks, 1):
    quote = block.replace('\n> ', '')
    print(f'Unit {i}: {len(quote)} chars')
# 若 Unit N 显示 >1000 chars 且后续 units 序号全部错位 → 格式损坏级联
```

**修复**：
- **立即修复**：找到格式损坏点，重新对齐 R/I 结构，不要逐个修改单元
- 若某 unit 的 R 段引文实际只有书名注脚（如 `"书名"——书名`），说明原始内容就是摘要式引用——需重写为基于 I-section 内容反推的核心论点（R 段只留最核心一句话，让 I 段承接推论细节）
- 修复后重跑 validator 确认所有 units 的 R-block 长度均 ≤150

**预防**：写完 SUMMARY.md 后立即用上述 Python 脚本逐个检查 R-block 长度，超限当场修，不要等 validator 报 failed 再回头。
