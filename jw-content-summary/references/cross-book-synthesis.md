# 跨书融合写作模式

当用户不是要总结单本书，而是要求“结合我已经总结过的多本书内容，再结合少量联网搜索，生成新的立场文件/人格文件/方法论宣言”时，使用本模式。

本模式把 jw-content-summary 当作“书摘二次编排器”，不机械套用单书三阶段流水线。

## 适用触发词

- “结合我对 A/B/C 的总结”
- “基于我之前整理的几本书，生成一个 X.md”
- “把这些书的思想熔成一个原则文件/人格文件/方法论文件”
- 输出目标是 `SOUL.md` / `PRINCIPLES.md` / `MANIFESTO.md` / `OPERATING_SYSTEM.md`

## 简化流程

1. 先找本地 SUMMARY.md / candidates
   - 优先在 `books/` 下搜索相关作者、书名、关键词。
   - 如果用户明确说“我总结过”，默认先找本地已有 SUMMARY.md，不要先追问原书文本。
2. 抽取可迁移原则，而不是重复摘要
   - 只抽：核心主旨、独特洞见、原则、边界、步骤、风险提醒。
   - 禁止把多本书重新各讲一遍；目标是融合成统一的行为/认知系统。
3. 建立主辅结构
   - 如果用户指定“以 A 为主，B/C 为辅”，输出结构必须体现主次。
   - 主轴思想来自主书系；工具层/校准层来自辅书系。
4. 少量联网只做校准，不喧宾夺主
   - 联网搜索只用于补充关键定义、权威表述或概念锚点。
   - 不得让网络材料压过用户自己的书摘材料。
5. 输出成可执行文档
   - 优先写成：总纲 / 核心底盘 / 方法论系统 / 决策协议 / 风险提示 / 风格约束。
   - 重点是“以后怎么做”，不是“这些书讲过什么”。

## 推荐结构模板

```markdown
# SOUL / PRINCIPLES / MANIFESTO

## 一句话总纲
## 核心底盘
## 方法论系统
## 决策协议
## 风险提示机制
## 风格与表达约束
## 自我约束清单
```

## 关键坑点

1. 不要误回到单本书摘要模式
   - 用户要的是融合后的“操作系统”，不是多本 SUMMARY 的拼盘。
2. 不要让联网结果盖过本地书摘
   - 网络内容只做定义校准和权威锚定。
3. 不要丢掉用户指定的主次关系
   - 例如“毛泽东思想为主，巴菲特/芒格为辅”必须在结构中显式体现。
4. 不要只写价值观口号
   - 必须落到可执行的决策步骤、边界、风险提示、表达风格。
