# 用课程/书籍内容补充分析类 Skill 指南

> 适用场景：用户有一批投资课程/书籍文件，要求补充到现有的分析类 skill（如 jw-company-analysis）。

## 工作流程

### 1. 内容分析阶段

读取所有课程文件，按主题分类：
- **估值方法**：PB/PEG/PS/PE 等估值工具
- **宏观分析**：货币政策、经济周期、行业趋势
- **风险管理**：反身性理论、投资者行为偏差、市场情绪
- **交易执行**：纪律体系、仓位管理、技术分析
- **行业分析**：行业分类、竞争格局、生命周期

### 2. 参考文件创建阶段

为每个主题创建独立的参考文件：
```
references/<topic>.md
```

**文件结构建议**：
```markdown
# 标题

> 来源：课程/书籍名称
> 适用场景：简要说明

## 一、核心概念
## 二、详细方法
## 三、实战案例
## 四、常见误区
## 五、实践清单
```

**内容要求**：
- 提炼核心方法论，不要照搬原文
- 保留关键数据和阈值（如 PB<0.8 买入、PEG<1.2 有价值）
- 包含实战案例和反面教材
- 标注适用行业和不适用行业

### 3. SKILL.md 更新阶段

在 SKILL.md 的相应 Step 中添加引用：

```markdown
> **📚 主题名称**：详见 `references/<topic>.md`，包含：
> - 要点1
> - 要点2
> - 要点3
```

**引用位置选择**：
- 估值方法 → Step 6 估值与安全边际
- 宏观分析 → Step 0.5 宏观环境评估
- 风险管理 → Step 7 风险分析与误判防护
- 交易执行 → Step 9 技术面分析
- 行业分析 → Step 2 行业分析

### 4. 质量验证阶段

检查：
- [ ] 所有参考文件已创建
- [ ] SKILL.md 中的引用路径正确
- [ ] 参考文件内容提炼准确
- [ ] 关键数据和阈值保留完整
- [ ] 适用/不适用行业标注清晰

## Pitfall 1：不要过度引用

SKILL.md 中的引用应简洁（3-5 行），详细内容放在参考文件中。避免在 SKILL.md 中重复大段参考文件内容。

## Pitfall 2：保持 Skill 的独立性

参考文件是补充，不是替代。SKILL.md 本身仍应包含核心框架和关键判断标准。参考文件提供"深入了解"的路径。

## Pitfall 3：课程内容的时效性

投资课程内容可能有时效性（如政策、市场数据）。在参考文件中标注数据截止日期，提醒用户注意更新。

## Pitfall 4：避免重复

补充前先检查 SKILL.md 和现有参考文件是否已有类似内容。避免创建重复文件。

## 示例：补充估值方法

**来源文件**：PB.pptx（PB估值法课程）

**创建文件**：`references/pb-valuation-guide.md`
- PB 基本概念和公式
- 六大行业 PB 区间详解
- PB 四步判断框架
- PB 三大常见误区
- 量能校准

**更新 SKILL.md**：在 Step 6.1 相对估值法部分添加引用
```markdown
> **📚 PB估值法**：详见 `references/pb-valuation-guide.md`
> - 适用于银行、地产、保险、券商、钢铁、煤炭等资产驱动型或强周期型行业
```
