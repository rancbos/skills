# jw-ebook-organizer

自动将电脑上散乱的电子书文件按学科分类整理为有序的本地图书馆。

`jw-ebook-organizer` 是基于 CLC（中国图书馆图书分类法）的本地电子书管理工具。

---

## 功能

将指定目录中的电子书（PDF/EPUB/MOBI/AZW3）自动按 **中国图书馆图书分类法（CLC/中图法）** 归入 29 个多级类目，同时执行：

- **多格式合并** — 同一本书的 epub/pdf/mobi 自动归入 `{书名-作者}` 子文件夹
- **作者归档** — 同一作者的 2+ 本书自动归入 `{作者}` 子文件夹
- **智能去重** — 同名同大小文件仅保留一份
- **版本标注** — 同名不同版本自动加 `_1`, `_2` 后缀
- **增量扫描** — 后续新增书籍只处理新文件，不重复全量
- **安全回滚** — 整理记录保存在 `.move_manifest.json`，可一键 `undo`
- **复制模式** — `--copy` 先复制再标记源文件，双保险
- **图书索引** — 自动生成 Markdown 格式的 `图书索引.md`

## 适用场景

- 电脑上积压了大量杂乱命名的电子书，想按学科分类整理
- 同一本书有多个格式版本（epub/pdf/mobi），不想重复占用空间
- 想快速知道自己的书库里有什么、在哪
- 经常添加新书，需要增量整理

## 快速开始

```bash
# 1. 扫描目标目录
python ebook_ops.py scan "F:\我的电子书" -o scan_result.json

# 2. 清洁书名
python ebook_ops.py clean-titles scan_result.json -o cleaned_result.json

# 3. 联网查分类 → WorkBuddy 逐个搜索书名，确定 CLC 分类 + 作者
#    输出 classify_plan.json

# 4. 预览整理结果
python ebook_ops.py move classify_plan.json --dry-run

# 5. 执行整理（复制模式更安全）
python ebook_ops.py move classify_plan.json --copy

# 6. 生成图书索引
python ebook_ops.py index "F:\我的电子书" --output "图书索引.md"

# 7. 不满意就回滚
python ebook_ops.py undo --dry-run
python ebook_ops.py undo
```

## 子命令一览

| 命令 | 功能 |
|------|------|
| `scan` | 扫描目录中的电子书，支持 `--incremental` 增量模式 |
| `clean-titles` | 从扫描结果中清洗出干净书名（去作者/来源/版本噪音） |
| `move` | 按分类计划移动文件，支持 `--dry-run` 预览和 `--copy` 复制模式 |
| `index` | 生成 Markdown 格式的图书索引目录 |
| `undo` | 撤销上一次整理，恢复文件到原始位置 |
| `clean-source` | 清理 `--copy` 模式产生的 `.source` 标记文件 |
| `report` | 生成整理报告（JSON） |
